شرح الفصل الأول: Getting Started (الْبَدْء)

الفصل الأول بيبدأ بتعريف بأصل ووردبريس، وبيغطي الأفكار الفلسفية لطريقة البناء، وكمان بيدخل في التفاصيل المهمة المتعلقة بقواعد كتابة كود PHP في سياق تطوير الثيمات.

1. مقدمة عن ووردبريس ودوره

الفصل بيوضح تاريخ الووردبريس:

  • البداية والتطور: ووردبريس اتعمل سنة 2003 على إيد مات مولينويج ومارك ليتل. الفكرة كانت إنه يكون نظام للنشر الشخصي يكون معماري وأنيق. بدأ كفرع من نظام قديم اسمه b2/cafelog.
  • أشهر CMS: ووردبريس حالياً هو أشهر نظام لإدارة المحتوى (CMS) على الويب، وأكتر من 60% من المواقع اللي بتستخدم أنظمة إدارة محتوى بتعتمد عليه.
  • التحول الكامل لـ CMS: بعد إصدار نسخة 3.0 (Thelonious) في 2010، اللي قدمت أنواع المنشورات المخصصة (custom post types) والقوائم المخصصة، زاد استخدام ووردبريس كنظام إدارة محتوى متكامل.
  • المجتمع (Community): ووردبريس بيعتبر مشروع بيحركه المجتمع (community-driven) لأنه مفتوح المصدر. أي حد يقدر يشارك في تطوير الكود الأساسي، أو التوثيق (WordPress Codex)، أو حتى في تطوير الإضافات والثيمات المتاحة.

2. تجهيز البيئة والتعمق في الثيمات

الفصل بيلمح على الحاجات اللي محتاجها عشان تبدأ:

  • تنصيب ووردبريس: الكتاب بيشرح إنه بعد ما تكون جهزت السيرفر المحلي (زي AMPs)، عملية تنصيب ووردبريس سهلة جداً وبتسمى "Famous 5-Minute Install".
  • نقاش البناء: بيفتح الفصل نقاش مهم بين بناء الثيم من الصفر (from Scratch) أو الاعتماد على حلول جاهزة زي Frameworks (أطر العمل).
    • مميزات الحلول الجاهزة: أطر العمل (زي Twitter Bootstrap) ممكن تخلي إنتاج النماذج الأولية (prototypes) سريع جداً.
    • رأي الكاتب: يفضل البناء من الصفر أو باستخدام أساس أنت اللي مطوره بنفسك. ده لأن الاعتماد على الحلول الجاهزة في كود الإنتاج (production code) ممكن يعمل مشاكل زي الـ Markup الضعيف أو صعوبة الصيانة بعد كده.

3. أساسيات الـ PHP وقواعد كتابة الأكواد

ده جزء مهم جداً في الفصل، وبيركز على القواعد اللي لازم يلتزم بيها المطور عشان يكتب كود متوافق مع ووردبريس ويكون مقروء (Readability).

أ. البنية البديلة لجمل التحكم (Alternate Syntax)

في تطوير ثيمات ووردبريس، هتلاقي المطورين بيستخدموا طريقة مختلفة لكتابة جمل التحكم زي if و while، وده بيحسن قراءة الملفات اللي فيها مزيج من PHP و HTML.

  • الطريقة العادية (اللي بتستخدم الأقواس المعقوفة): 
    if( condition ) {
    // do something
}
  • الطريقة البديلة (WordPress Style): بنستخدم نقطتين رأسيتين (الكولون :) بدلاً من القوس الافتتاحي، وكلمة نهاية للجملة (زي endif; أو endwhile;) بدلاً من القوس الإغلاقي.

مثال على البنية البديلة (كود):

if( condition ):
    // do something
endif;

مثال على استخدامها في الحلقة (The Loop) من ثيم Twenty Twelve:

<?php if ( have_posts() ) : ?>
    <?php /* Start the Loop */ ?>
    <?php while ( have_posts() ) :
        the_post(); ?>
        <?php get_template_part( 'content', get_post_format() ); ?>
    <?php endwhile; ?>
    <?php twentytwelve_content_nav( 'nav-below' ); ?>
<?php else : ?>
    <h1 class="entry-title"><?php _e( 'Nothing Found', 'twentyeleven' ); ?></h1>
<?php endif; endwhile; ?>

ملحوظة مهمة: الكاتب بيأكد إن البنية البديلة دي المفروض تستخدم بشكل أساسي في ملفات القوالب اللي بتطلع HTML. لكن لو بتكتب كود في ملف functions file (ملف الوظائف) أو كود ملوش علاقة مباشرة بالـ HTML، الأفضل إنك تستخدم البنية القياسية اللي فيها الأقواس المعقوفة.

ب. قاعدة يودا (Yoda Condition)

دي قاعدة ممتازة عشان تخلي الكود بتاعك نظيف وآمن من الأخطاء السهلة. القاعدة بتلزمك إنك تخلي المتغير اللي بتقارنه في الجهة اليمين من جملة الشرط.

ليه بنستخدمها؟ لو نسيت علامة يساوي واحدة بالغلط (مثلاً كتبت = بدل ==)، الكود هيطلع خطأ (parse error) بدلاً من إنه يرجع قيمة true بالغلط ويتسبب في مشكلة صعبة تكتشفها.

مثال على قاعدة يودا (كود):

if ( true == $the_force ) {
    $victorious = you_will( $be );
}

ج. المتغيرات والأشياء (Objects and Variables)

بيتم تقديم مفهوم الـكلاسات والأشياء (Objects and Classes)، وبتبدأ تتعرف على كلاس مهم جداً في ووردبريس وهو WP_Query. ده الكلاس المسؤول عن استرجاع المحتوى.

مثال على استدعاء ميثود من كلاس WP_Query:

$new_query->the_post();

هتتعمق في فهم WP_Query أكتر في الفصل الثالث عشان تعرف إزاي تجيب المحتوى لملفات القوالب.

د. قواعد التسمية (Naming Conventions)

في ووردبريس، لازم تلتزم بقواعد تسمية صارمة. كل الكلمات في أسماء الدوال والمتغيرات لازم تتفصل بـ Underscores (الشرطة السفلية _) في كل الأوقات.

أهلاً بك يا فندم، طلبك على عيني وراسي. الفصل التاني ده هو اللي بيعرفك على "خريطة الشغل" أو الهيكل العظمي (Anatomy) بتاع الثيم بتاعك، وإزاي ووردبريس بيقرر هيعرض إيه وإزاي.

ده شرح مستفيض للفصل الثاني "Theme Anatomy and Template Hierarchy" (تشريح الثيم وتسلسل القوالب) بالهجة المصرية، مع الأمثلة والأكواد:

شرح الفصل الثاني: تشريح الثيم وتسلسل القوالب

الفصل التاني ده يعتبر هو البوصلة بتاعتك في عالم ووردبريس. بيشرحلك إيه هي الملفات الأساسية اللي مينفعش الثيم بتاعك يشتغل من غيرها، وإيه الدوال (Functions) اللي لازم تكون موجودة، والأهم من كده، بيشرحلك إزاي ووردبريس بيفكر عشان يعرض الصفحة اللي بيطلبها المستخدم.

1. الملفات الأساسية المطلوبة (Bare Minimum Files)

يا فندم، أي ثيم ووردبريس عشان يشتغل لازم يكون فيه ملفين على الأقل:

  1. style.css: ملف الـ CSS اللي بيحدد شكل الثيم.
  2. index.php: ده الملف اللي بيعرض المحتوى الأساسي (المحتوى الاحتياطي أو الـ Fallback).

أ. ملف الـ style.css (بطاقة تعريف الثيم)

ملف الـ style.css مش بس للستايل، ده كمان بطاقة تعريف الثيم. لازم يكون فيه تعليق (Comment Block) في الأول بيحتوي على البيانات الأساسية اللي ووردبريس بيحتاجها عشان يتعرف على الثيم ويظهره في لوحة التحكم.

أهم البيانات اللي لازم تكون في التعليق:

  • Theme Name: اسم الثيم.
  • Author: اسم المطور.
  • Version: رقم إصدار الثيم.
  • Text Domain: مُعرّف فريد بيستخدم في عملية الترجمة (Internationalization)، وده بيكون غالبًا نسخة مصغرة من اسم الثيم.

مثال لكود الـ style.css Header:

/*
 * Theme Name: Pro WordPress Theme Development
 * Author: Adam Onishi
 * Description: An example theme for the Pro WordPress theme development book
 * Version: 0
 * Text Domain: prowordpress
 */

ملاحظة إضافية: عشان الثيم يظهر بشكل حلو في قائمة الثيمات، لازم تحط صورة اسمها screenshot.png (موصى بحجم 600x450px) في الجذر بتاع مجلد الثيم.

2. الملفات المشتركة الأساسية (Core Template Files)

عشان الثيم يبقى عملي، بتحتاج تقسمه لأجزاء مشتركة بتظهر في كل الصفحات، زي الهيدر والفوتر والسايدبار:

أ. header.php

الملف ده بيحتوي على بداية كود الـ HTML، من أول <!DOCTYPE html> لحد بداية جسم الصفحة <body>، وأيضاً وسم <header> بتاع الموقع.

أهم الدوال اللي لازم تحطها في ملف header.php:

  1. wp_head(): الدالة دي إلزامية ولازم تتحط قبل قفلة وسم الـ <head>. وظيفتها إنها بتسمح لووردبريس وللإضافات إنهم يضيفوا ملفات CSS أو أكواد Scripts أو Meta Tags في المكان الصح في الصفحة.
  2. language_attributes(): بتتحط في وسم الـ <html >.
  3. bloginfo('charset'): بتعرض الـ Encoding بتاع الموقع.
  4. wp_title(): بتعرض عنوان الصفحة.

مثال لكود في header.php:

<!DOCTYPE html>
<html class="no-js" <?php language_attributes(); ?>>
<head>
    <meta charset="<?php bloginfo( 'charset' ); ?>" />
    <meta name="viewport" content="width=device-width" />
    <title><?php wp_title( '|', true, 'right' ); ?></title>
    <!-- هنا لازم wp_head() عشان الإضافات تشتغل صح -->
    <?php wp_head(); ?>
</head>
<body <?php body_class(); ?>>
    <header class="site-header">
        <!-- هنا ممكن تعرض اللوجو زي المثال ده -->
        <h1><a href="<?php echo home_url(); ?>">Pro WordPress Theme Development</a></h1>
    </header>

ب. footer.php

الملف ده بيحتوي على نهاية الـ HTML (قفلة جسم الصفحة و الـ HTML).

أهم الدوال اللي لازم تحطها في ملف footer.php:

  1. wp_footer(): الدالة دي إلزامية ولازم تتحط قبل قفلة وسم الـ </body>. وظيفتها إنها بتضيف الأكواد اللازمة في نهاية الصفحة، زي أكواد الإحصائيات (Analytics) أو شريط الإدارة بتاع ووردبريس (Admin Bar) اللي بيظهر لما تكون مسجل دخول. لو نسيتها، ممكن بعض الإضافات ما تشتغلش صح.

ج. sidebar.php و comments.php

  • sidebar.php: يُستخدم لعرض المحتوى الجانبي المتكرر في صفحات كتير، زي الـ Widgets.
  • comments.php: يُستخدم لتوليد كود الـ HTML الخاص بنظام التعليقات في منشور معين.

3. التسلسل الهرمي للقوالب (Template Hierarchy)

هنا بتيجي النقطة الأهم في الفصل ده. إزاي ووردبريس بيقرر يستخدم ملف page.php ولا ملف single.php ولا index.php؟ بيستخدم نظام الأولوية (Priority).

القاعدة: ووردبريس بيبدأ يبحث عن الملف الأكثر تحديداً (Most Specific) لنوع المحتوى ده. لو مالقاهوش، بيرجع تدريجياً للملف الأعم، لحد ما يوصل لـ index.php اللي هو الـ Fallback النهائي.

نوع الصفحة (المحتوى) الملف الأكثر تحديداً (الأولوية) الملف الاحتياطي العام المرجع
صفحة ثابتة (مثل صفحة "من نحن") page-{slug}.php أو page-{ID}.php page.php > index.php
منشور فردي (بوست عادي) single-post.php (لنوع المنشور الافتراضي) single.php > index.php
منشور نوع مخصص (Custom Post Type) single-{posttype}.php (زي single-movie.php) single.php > index.php
أرشيف عام (تصنيفات، وسوم، مؤلف) category.php، tag.php، date.php، إلخ archive.php > index.php
صفحة 404 (غير موجودة) 404.php index.php (افتراضياً)

ملحوظة عن single.php: ملف single.php بيستخدم لعرض كل المنشورات الفردية، بما في ذلك أنواع المنشورات المخصصة (Custom Post Types)، لو مفيش ملف محدد باسم single-posttype.php.

4. تضمين الأصول (Enqueuing Assets)

لتحميل ملفات CSS (الـ Styles) وملفات JavaScript (الـ Scripts)، مينفعش تحطها مباشرة في ملف الـ header.php باستخدام وسوم <link> و <script>. الصح إنك تستخدم وظائف ووردبريس اللي اسمها Enqueuing عشان تتأكد إن الملفات بتتحمل مرة واحدة، في المكان الصح، وبالترتيب الصح.

أ. تحميل Styles و Scripts:

وظائف التحميل دي بتتم في ملف functions.php.

  • لتحميل Style: استخدم wp_enqueue_style().
  • لتحميل Script: استخدم wp_enqueue_script().

مثال (في ملف functions.php) لإزالة jQuery الخاص بووردبريس وتحميل نسخة من Google API:

الكاتب بينصح تعمل كده عشان تحسن الأداء (Caching)، بس بشرط ما تعملش كده في لوحة تحكم الأدمن:

function prowordpress_scripts() {
    // نتأكد إننا مش في لوحة التحكم عشان ما نبوظش اعتمادية ووردبريس
    if ( ! is_admin() ) {
        // 1. نشيل تسجيل نسخة ووردبريس الأصلية من jQuery
        wp_deregister_script('jquery');

        // 2. نسجل ونحمل النسخة اللي عايزينها من Google CDN
        wp_enqueue_script('jquery', 'http://ajax.googleapis.com/ajax/libs/jquery/1/jquery.min.js', false, null, true);
    }
}
// نضيف الدالة دي على الـ hook بتاع تحميل الأكواد
add_action('wp_enqueue_scripts', 'prowordpress_scripts');

5. الدوال الشرطية (Conditional Tags)

الدوال الشرطية بتسمحلك تعرف نوع الصفحة اللي بيتم عرضها عشان تقدر تعمل تخصيص (Customization). دي مهمة جداً خصوصاً في ملفات الـ Fallback زي index.php.

أمثلة بسيطة للدوال الشرطية:

  • is_home(): هل دي الصفحة الرئيسية (اللي بتعرض آخر المنشورات)؟
  • is_page(): هل دي صفحة ثابتة؟
  • is_single(): هل ده منشور فردي؟

مثال لاستخدام الدوال الشرطية لتحديد نوع الأرشيف وعرض عنوان مخصص:

<?php if ( is_category() ) : ?>
    <h1>Archive for: <?php single_cat_title(); ?></h1>
<?php elseif ( is_tag() ) : ?>
    <h1>Posts Tagged: <?php single_tag_title(); ?></h1>
<?php elseif ( is_day() ) : ?>
    <h1>Archive for <?php the_time('F jS, Y'); ?></h1>
<?php else : ?>
    <h1>Archives</h1>
<?php endif; ?>

6. بناء الثيم، الجزء الأول (Building Your Theme, Part 1)

في نهاية الفصل، بتبدأ تطبيق كل اللي اتعلمته في بناء الهيكل الأساسي للثيم. ده بيتم عبر إنشاء الملفات الأساسية وتضمين الدوال المطلوبة في ملف functions.php (ملف الوظائف):

أ. دالة الإعدادات الأساسية (functions.php)

في الملف ده بتبدأ بتعريف دالة الإعدادات اللي بتخلي ووردبريس يضيف دعم لبعض الميزات:

if ( ! function_exists( 'prowordpress_setup' ) ) :

function prowordpress_setup() {

    // تفعيل دعم لروابط تغذية RSS
    add_theme_support( 'automatic-feed-links' );

    // تفعيل دعم للصور المصغرة (Post Thumbnails)
    add_theme_support( 'post-thumbnails' );

    // (ممكن نضيف دعم للقوائم هنا في فصول جاية)
}

endif; // نهاية prowordpress_setup

// ربط الدالة بالـ Hook اللي بيشتغل بعد إعداد الثيم
add_action( 'after_setup_theme', 'prowordpress_setup' );

الـ Hook اللي اسمه after_setup_theme هو المكان المظبوط عشان تبدأ تضيف فيه ميزات للثيم.
