Ahlan! This is Chapter 1: Your first Vue.js Web Application. This chapter is your starting point, where you will learn all the basic tools (the core fundamentals) you need to build dynamic web pages using Vue.js.

In this chapter, you build a simple voting application called "UpVote!" that looks like social news websites. This app displays a list of submissions sorted by the number of votes, and users can click an icon to increase a submission's vote count, making the list instantly re-sort.

Step 1: Getting Started

We need a simple setup to start learning Vue quickly. For this first step, we use a very simple setup where we load Vue directly from a link on the internet (a Content Delivery Network, or CDN) inside an HTML file. This lets us run the app immediately by just opening the HTML file in the browser.

Your application needs a place to live on the web page. You must define a space (an HTML element) that your Vue application will control. This space is called the view. In the example app, this is done with a <div> that has the ID app.

Code Example (index.html):

<div id="app">
  <!-- This is the space Vue controls -->
  <h2 class="title has-text-centered dividing-header">UpVote!</h2>
</div>
<script src="https://unpkg.com/vue"></script>
  • id="app": This tag marks the spot where the Vue application will be mounted (attached).
  • <script src="...">: This line loads the main Vue library from the CDN, making the special Vue function available for us to use.

Step 2: The Vue Instance (The Brain)

Every Vue application must have a starting point or a "brain." This starting point is called the Vue instance. It is created using the special Vue function, and you give it options like where to attach itself and what data it holds.

You must tell the Vue instance which part of the HTML it controls. You use the el option (short for "element") to point the Vue instance to the container you defined in the HTML (the <div id="app">).

Code Example (main.js):

new Vue({
  el: '#app' // 1. Tell Vue to control the element with ID 'app'
});
  • new Vue({...}): This creates the brain of your application.
  • el: '#app': This option tells the Vue instance to mount (attach) to the HTML element that has the ID app.

Step 3: Making the View Data-Driven

We store all the information that the app needs inside the Vue instance. The information is stored in a data object inside your Vue instance. This is how you link your JavaScript variables directly to what the user sees on the screen.

Code Example (main.js): Imagine you have a list of product ideas in a JavaScript variable called Seed.submissions.

new Vue({
  el: '#app',
  data: {
    // 2. Put your application data here:
    submissions: Seed.submissions
  }
});
  • data: {...}: This is where you specify the data that the Vue instance will manage.
  • submissions: Seed.submissions: We create a property named submissions inside the Vue instance and give it the list of items from our sample data.

Step 4: Data Binding (Connecting Data to the Screen)

Data binding means connecting your stored information directly to the HTML display. When the data changes in your Vue instance, the part of the HTML connected to it changes automatically (it is reactive).

4a. Showing Text Data (Mustache Syntax)

To show text information on the screen, use the "Mustache" syntax: double curly braces. The double curly braces {{ }} look like a mustache, and Vue replaces everything inside them with the actual value of your data property.

  • Practical Example: If you want to show the title of the first item in your submissions list, you write {{ submissions.title }} in your HTML.

4b. Binding to HTML Attributes (v-bind)

To connect data to attributes inside an HTML tag (like the link of an image or a link URL), you must use the v-bind directive. The v-bind directive is a special Vue instruction that tells Vue to connect a data value to an HTML attribute (like src for an image or href for a link).

  • Practical Example: If you want to connect the image URL (submissionImage) of your first item to the src attribute of an <img> tag, you write: <img v-bind:src="submissions.submissionImage">.
Full Syntax Shorthand (Quick Way)
v-bind:src="..." :src="..."
v-on:click="..." @click="..."

Step 5: List Rendering (Showing Many Items)

If you have a list of many things (like all the submissions), you must repeat the same design block for each item. You use the special Vue instruction v-for to repeat an element or component for every item in a data list.

  • Analogy: Think of v-for as a photocopier for your HTML. You design one copy (the template), and v-for prints out a copy for every piece of data in your list.

Code Example: To repeat the design block (<article class="media">) for every submission object in your submissions list:

<article v-for="submission in submissions" class="media">
  <!-- Content for one submission goes here -->
  <h1>{{ submission.title }}</h1>
  <img :src="submission.submissionImage">
</article>
  • v-for="submission in submissions": This instruction loops through the array named submissions and temporarily names the current item submission.
  • Inside the loop, you use submission.title instead of submissions.title.

Important Tip: The key Attribute When using v-for, you should always add a :key attribute with a unique ID for every repeated item (like submission.id). This helps Vue keep track of each item efficiently, especially when the list changes order.

<article v-for="submission in submissions" v-bind:key="submission.id" class="media">
  <!-- ... -->
</article>

Step 6: Computed Properties (Smart, Automatic Calculations)

Computed properties are special functions that automatically calculate and return a value based on your data. If the underlying data changes, the computed property automatically runs again and updates the display instantly. This is perfect for sorting, filtering, or complex mathematical tasks in your application.

  • Practical Example (Sorting): In the UpVote! app, submissions must always be sorted by the number of votes, highest first.

Code Example (main.js): We define sortedSubmissions inside the computed property of the Vue instance.

new Vue({
  // ... data object is here
  computed: {
    sortedSubmissions () {
      // 1. We get the original list: this.submissions
      return this.submissions.sort((a, b) => {
        // 2. We sort it by votes (b.votes - a.votes sorts highest to lowest)
        return b.votes - a.votes
      });
    }
  }
});

Then, we update the v-for loop in the HTML to use the sorted list instead of the original one:

<article v-for="submission in sortedSubmissions" v-bind:key="submission.id" class="media">
  <!-- ... -->
</article>

Now, Vue automatically uses the sorted list, and if any submission's votes change, Vue automatically re-runs this function and updates the list order on the screen.

Step 7: Event Handling (Making the App Interactive)

To respond to user actions (like a click), you use the v-on directive, or the shorthand @. The v-on directive listens for events in the DOM (like click, mouseover, submit) and runs a JavaScript method when that event happens.

  • Practical Example (Up-voting): We want a method called upvote to run when the user clicks the arrow icon.

Code Example (HTML Template): We attach the event listener directly to the element the user clicks:

<span class="icon is-small" v-on:click="upvote(submission.id)">
  <i class="fa fa-chevron-up"></i>
</span>
  • v-on:click: This tells Vue to listen for a click event.
  • "upvote(submission.id)": When the click happens, run the method named upvote and pass the unique ID of the current item to it.

Code Example (Vue Instance - main.js): You define the upvote function inside the methods property of the Vue instance.

new Vue({
  // ... data and computed properties here
  methods: {
    upvote(submissionId) {
      // 1. Find the item that matches the ID passed in
      const submission = this.submissions.find(
        submission => submission.id === submissionId
      );
      // 2. Increase its vote count by 1.
      submission.votes++;
    }
  }
});

When submission.votes increases, Vue automatically detects the change, and the sortedSubmissions computed property re-runs, instantly updating the list order on the screen. This is called reactive state.

Step 8: Components (Building Blocks)

For large applications, putting everything in one big file is messy. Components help you organize your code into small, reusable pieces. A component is an isolated module that groups together its own design (HTML), behavior (JavaScript logic), and sometimes styling (CSS). This makes the code easier to maintain and reuse.

  • Analogy: If your whole app is a building, the main Vue instance is the blueprint for the whole building, but components are the standardized parts, like windows, doors, or entire standardized apartments, which you can use many times.

Parent-Child Relationship In the UpVote! app, the main Vue instance (the new Vue({}) part) becomes the parent, and the individual submission item becomes a child component.

1. Define the Child Component (main.js): We define the component logic and template separately:

const submissionComponent = {
  // The design (HTML) for ONE submission item
  template: `<div>... all the HTML for one submission ...</div>`,

  // What data this component receives from its parent
  props: ['submission', 'submissions'],

  // The methods that run when this component is interacted with
  methods: {
    upvote(submissionId) {
      // ... upvote logic using this.submissions ...
    }
  }
};
  • template: Contains the HTML markup for a single submission item.
  • Props: This is how the parent component sends data down to the child component. The child must explicitly declare the data names it expects (here: submission details and the main submissions list).

2. Register the Child Component (main.js): You register the child component inside the parent Vue instance so the parent knows about it:

new Vue({
  el: '#app',
  // ... data and computed properties here
  components: {
    'submission-component': submissionComponent
  }
});

3. Use the Child Component (HTML Template): In the parent's HTML, you use the component like any other HTML tag inside your v-for loop:

<article v-for="submission in sortedSubmissions" v-bind:key="submission.id" class="media">
  <submission-component
    v-bind:submission="submission"
    v-bind:submissions="sortedSubmissions">
  </submission-component>
</article>
  • v-bind:submission="submission": This passes the current item data (submission) down as a prop named submission to the child component.

Summary and Practical Advice

Short Chapter Summary

Chapter 1 introduces the core fundamentals of building a Vue application by creating the UpVote! voting app. Key ideas include defining the Vue instance (new Vue) and mounting it to the DOM (el: '#app') to control a view. We learned data binding using {{ }} for text and v-bind (or :) for attributes, and how v-for renders lists. Finally, we implemented interactivity with v-on (or @) to run methods and used components and props to organize the application into reusable blocks.

Main Key Ideas to Remember

  1. The Vue Instance is the Start: Everything begins with new Vue() controlling an area of your HTML (el).
  2. Reactivity is Automatic: When data in your data property changes, the HTML that is bound to it updates instantly.
  3. Use Computed Properties for Logic: Use computed properties for any calculation (like sorting or filtering) that depends on existing data, as they update automatically.
  4. Components and Props: Break your app into small components for organization, and always pass data down from parent to child using props.
