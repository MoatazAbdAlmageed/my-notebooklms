🎯 1. ما هي Vue.js؟

Vue.js (تنطق “فيو”) هي إطار عمل (JavaScript Framework) مفتوح المصدر لبناء واجهات المستخدم (User Interfaces) وتطبيقات الويب التفاعلية.

تتيح لك Vue بناء واجهات أمامية مكوّنة من مكونات مستقلة (Components) — كل مكون يحتوي على منطق الواجهة (JavaScript) والقالب (HTML) والتنسيق (CSS) في ملف واحد من نوع .vue.

تم إنشاء Vue بواسطة Evan You عام 2014، وهدفه الأساسي كان:

"دمج أفضل الأفكار من Angular وReact في إطار خفيف وسهل التعلم."

⚖️ 2. مقارنة Vue.js مع React وAngular

المقارنة Vue.js React Angular
نوع الأداة إطار (Framework) مكتبة (Library) إطار كامل (Full Framework)
طريقة الربط HTML + توجيهات JSX TypeScript
منحنى التعلم سهل وسلس متوسط صعب نسبياً

الحجم صغير متوسط كبير
استخدام مع Laravel ممتاز وسلس جيد معقد

النتيجة:
Vue.js ممتازة للمشاريع الصغيرة والمتوسطة، كما أنها الأنسب لمطوري Laravel بفضل سهولة الدمج والتكامل بينهما.

⚙️ 3. كيف تعمل Vue.js (نظام التفاعل – Reactivity System)

قلب Vue.js يعتمد على ما يُعرف باسم نظام التفاعل (Reactivity System).
أي أن Vue تتابع التغيرات في البيانات بشكل ذكي وتُحدّث واجهة المستخدم تلقائيًا.

مثال بسيط:

<div id="app">
  <p>عدد النقرات: {{ count }}</p>
  <button @click="count++">اضغط هنا</button>
</div>

<script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>
<script>
  const { createApp } = Vue

  createApp({
    data() {
      return {
        count: 0
      }
    }
  }).mount('#app')
</script>

📍 شرح الكود:

  • {{ count }}: تُستخدم لعرض قيمة متغير داخل القالب (Interpolation).
  • @click: تعني v-on:click، أي “نفّذ هذا عند الضغط”.
  • عندما تتغير قيمة count → يتم تحديث النص في <p> تلقائيًا بدون أي كود إضافي.

  • main.js: هو الذي يقوم بتهيئة التطبيق وربطه بعنصر DOM رئيسي مثل #app.

⚙️ 1. إنشاء أول تطبيق Vue

لنبني تطبيق بسيط يعرض رسالة ترحيب ويغيّرها عند الضغط على زر.

<div id="app">
  <h1>{{ message }}</h1>
  <button @click="changeMessage">غيّر الرسالة</button>
</div>

<script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>
<script>
  const { createApp } = Vue

  createApp({
    data() {
      return {
        message: 'مرحباً بك في Vue.js'
      }
    },
    methods: {
      changeMessage() {
        this.message = 'تم تغيير الرسالة!'
      }
    }
  }).mount('#app')
</script>

📘 الشرح:

  • {{ message }} تُستخدم لعرض المتغير في القالب.
  • @click اختصار لـ v-on:click.
  • this.message تعني الوصول إلى المتغير داخل data().
  • عند الضغط على الزر يتم استدعاء الدالة changeMessage().

🧩 2. التصريحات (Directives)

التصريحات هي أوامر خاصة تبدأ بـ v- وتتحكم في عرض العناصر أو سلوكها داخل القالب.

🔹 v-bind

تستخدم لربط خاصية HTML مع متغير من البيانات.

<div id="app">
  <img v-bind:src="imageUrl" alt="صورة Vue">
</div>

<script>
  const { createApp } = Vue
  createApp({
    data() {
      return {
        imageUrl: 'https://vuejs.org/images/logo.png'
      }
    }
  }).mount('#app')
</script>

يمكنك استخدام الاختصار : بدلاً من v-bind:
👉 <img :src="imageUrl">

🔹 v-if و v-else و v-show

تُستخدم للتحكم في ظهور العناصر بناءً على شرط.

<div id="app">
  <p v-if="isLoggedIn">مرحباً {{ username }}</p>
  <p v-else>الرجاء تسجيل الدخول</p>

  <button @click="toggleLogin">تبديل الحالة</button>
</div>

<script>
  const { createApp } = Vue
  createApp({
    data() {
      return {
        isLoggedIn: false,
        username: 'معتز'
      }
    },
    methods: {
      toggleLogin() {
        this.isLoggedIn = !this.isLoggedIn
      }
    }
  }).mount('#app')
</script>

💡 الفرق بين v-if و v-show:

  • v-if يضيف أو يزيل العنصر من DOM.
  • v-show يخفيه فقط باستخدام CSS (display: none).

🔹 v-for

لعرض قائمة من العناصر.

<div id="app">
  <ul>
    <li v-for="(item, index) in items" :key="index">
      {{ index + 1 }} - {{ item }}
    </li>
  </ul>
</div>

<script>
  const { createApp } = Vue
  createApp({
    data() {
      return {
        items: ['Vue.js', 'React', 'Angular']
      }
    }
  }).mount('#app')
</script>

📘 :key ضروري ليعرف Vue كيف يميز العناصر أثناء التحديث.

⚡ 3. التعامل مع الأحداث (Event Handling)

استخدام v-on أو @ للتعامل مع أحداث المستخدم مثل النقر أو الكتابة.

<div id="app">
  <input @keyup.enter="sayHello" placeholder="اكتب ثم اضغط Enter">
</div>

<script>
  const { createApp } = Vue
  createApp({
    methods: {
      sayHello() {
        alert('مرحباً بك!')
      }
    }
  }).mount('#app')
</script>

يمكنك تحديد نوع الحدث بعد النقطة مثل:

  • @keyup.enter
  • @keyup.esc
  • @submit.prevent (لمنع إعادة تحميل الصفحة عند الإرسال)

🔄 4. البيانات (data) والطرق (methods)

كل تطبيق Vue يحتوي على كائنين أساسيين:

القسم الوظيفة
data() يحتوي على القيم والمتغيرات المستخدمة داخل القالب
methods يحتوي على الدوال التي تنفذ المنطق

مثال:

<div id="app">
  <h2>{{ fullName }}</h2>
  <button @click="reverseName">اعكس الاسم</button>
</div>

<script>
  const { createApp } = Vue
  createApp({
    data() {
      return {
        fullName: 'Moataz Mohammady'
      }
    },
    methods: {
      reverseName() {
        this.fullName = this.fullName.split('').reverse().join('')
      }
    }
  }).mount('#app')
</script>

⚙️ 5. الحقول المحسوبة (Computed Properties)

تُستخدم عندما تريد إنشاء قيمة مشتقة من بيانات أخرى بشكل تلقائي.

<div id="app">
  <input v-model="firstName" placeholder="الاسم الأول">
  <input v-model="lastName" placeholder="الاسم الأخير">
  <p>الاسم الكامل: {{ fullName }}</p>
</div>

<script>
  const { createApp } = Vue
  createApp({
    data() {
      return {
        firstName: '',
        lastName: ''
      }
    },
    computed: {
      fullName() {
        return `${this.firstName} ${this.lastName}`
      }
    }
  }).mount('#app')
</script>

💡 ميزة computed:

  • تُحدّث فقط عندما تتغير البيانات المرتبطة بها.
  • تُخزَّن مؤقتًا (Cached) لتحسين الأداء.

👀 6. المراقبون (Watchers)

تسمح لك بمراقبة أي متغير وتنفيذ كود عند تغيّره.

<div id="app">
  <input v-model="age" placeholder="ادخل عمرك">
  <p>{{ message }}</p>
</div>

<script>
  const { createApp } = Vue
  createApp({
    data() {
      return {
        age: 0,
        message: ''
      }
    },
    watch: {
      age(newValue) {
        if (newValue < 18) this.message = 'أنت تحت السن القانوني.'
        else this.message = 'مرحباً بالبالغين!'
      }
    }
  }).mount('#app')
</script>

📘 الفرق بين watch و computed:

  • computed: لإرجاع قيمة مشتقة.
  • watch: لتنفيذ إجراء عند تغيّر قيمة معينة (مثل طلب API أو التحقق من صحة).

⚙️ 1. ما هي المكونات؟

المكون (Component) هو كائن (Object) له:

  • بيانات (data)

⚙️ 4. التواصل بين المكونات (Events & Emit)

إذا كان المكون الابن يريد إرسال حدث إلى الأب، نستخدم this.$emit().

<div id="app">
  <counter @incremented="handleIncrement"></counter>
  <p>عدد النقرات الكلي: {{ total }}</p>
</div>

<script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>
<script>
  const { createApp } = Vue

  const Counter = {
    data() {
      return { count: 0 }
    },
    template: `
      <button @click="increment">عدد النقرات: {{ count }}</button>
    `,
    methods: {
      increment() {
        this.count++
        this.$emit('incremented') // 🔄 إرسال حدث للأب
      }
    }
  }

  createApp({
    data() {
      return { total: 0 }
    },
    methods: {
      handleIncrement() {
        this.total++
      }
    },
    components: { counter: Counter }
  }).mount('#app')
</script>

📘 الشرح:

  • المكون Counter يُصدر حدثًا عند كل نقرة.
  • المكون الأب يستمع لهذا الحدث باستخدام @incremented.

⚙️ الفصل الرابع: إدارة الحالة (State Management)

🎯 مقدمة

عندما يبدأ تطبيقك بالنمو، وتصبح هناك عدة مكونات تتشارك في نفس البيانات، ستواجه مشكلة:

“كيف أنقل البيانات من مكون إلى آخر بطريقة منظمة دون تعقيد؟”

في البداية يمكن تمرير البيانات عبر props و emit،
لكن عندما تصبح البيانات مشتركة بين عدة طبقات من المكونات — ستتحول هذه الطريقة إلى فوضى.

هنا يأتي دور إدارة الحالة (State Management)،
وأشهر أداة لذلك في Vue.js هي Vuex (في Vue 2 و 3) أو Pinia (في Vue 3 الحديثة).

في هذا الفصل سنشرح Vuex أولاً، لأنه يوضح مفهوم “إدارة الحالة” بعمق.

🧩 1. ما المقصود بـ “الحالة” (State)؟

الحالة (State) هي ببساطة “البيانات المشتركة” داخل تطبيقك.
مثلاً:

  • المستخدم المسجل حالياً
  • عناصر السلة (Cart Items)
  • إعدادات اللغة أو المظهر

بدلاً من حفظ هذه القيم في كل مكون على حدة، يتم وضعها في مكان مركزي واحد يستطيع الجميع الوصول إليه.

🔍 2. مشكلة تمرير البيانات في المكونات

افترض أن لدينا ثلاثة مكونات:

App.vue → ProductList → ProductItem

لو أردنا تعديل “عدد المنتجات في السلة” بعد الضغط على زر داخل ProductItem،
فعلينا تمرير الحدث من الابن إلى الأب ثم إلى المكون الرئيسي...
وهكذا تصبح العملية مرهقة كلما كبر المشروع.

🧱 3. ما هو Vuex؟

Vuex هو مكتبة رسمية من Vue لإدارة الحالة،
تعمل مثل قاعدة بيانات صغيرة داخل الواجهة (Frontend).

📦 Vuex يتكون من 4 أجزاء أساسية:

الجزء الوصف
state يحتوي على البيانات المركزية
getters دوال ترجع قيم مشتقة من state
mutations تغيّر البيانات بطريقة متزامنة
actions تنفّذ عمليات غير متزامنة (مثل استدعاء API) ثم تغيّر البيانات

📘 5. إعداد Vuex Store

📄 store/index.js

import { createStore } from 'vuex'

const store = createStore({
  state() {
    return {
      count: 0
    }
  },
  mutations: {
    increment(state) {
      state.count++
    }
  },
  actions: {
    incrementAsync({ commit }) {
      setTimeout(() => {
        commit('increment')
      }, 1000)
    }
  },
  getters: {
    doubleCount(state) {
      return state.count * 2
    }
  }
})

export default store

🧠 7. استخدام Vuex داخل المكونات

📄 App.vue

<template>
  <div class="counter">
    <h2>العداد: {{ count }}</h2>
    <p>القيمة المضاعفة: {{ doubleCount }}</p>

    <button @click="increment">زيادة فورية</button>
    <button @click="incrementAsync">زيادة بعد ثانية</button>
  </div>
</template>

<script>

export default {
  computed: {
    ...mapState(['count']),
    ...mapGetters(['doubleCount'])
  },
  methods: {
    ...mapMutations(['increment']),
    ...mapActions(['incrementAsync'])
  }
}
</script>

📘 الشرح:

  • mapState لجلب القيم من state.
  • mapGetters لجلب القيم المحسوبة.
  • mapMutations لتنفيذ تغييرات فورية.
  • mapActions لتنفيذ عمليات غير متزامنة.

🔄 8. مثال عملي صغير – سلة المشتريات (Shopping Cart)

📄 store/index.js

import { createStore } from 'vuex'

export default createStore({
  state() {
    return {
      cart: []
    }
  },
  mutations: {
    addToCart(state, product) {
      state.cart.push(product)
    },
    removeFromCart(state, id) {
      state.cart = state.cart.filter(p => p.id !== id)
    }
  },
  getters: {
    totalItems(state) {
      return state.cart.length
    },
    totalPrice(state) {
      return state.cart.reduce((sum, p) => sum + p.price, 0)
    }
  }
})

📄 Cart.vue

<template>
  <div>
    <h2>🛒 سلة المشتريات</h2>
    <p>عدد العناصر: {{ totalItems }}</p>
    <p>الإجمالي: {{ totalPrice }} ريال</p>

    <button @click="addToCart({ id: 1, name: 'تمر سكري', price: 35 })">
      أضف تمر سكري
    </button>
  </div>
</template>

<script>
import { mapGetters, mapMutations } from 'vuex'

export default {
  computed: {
    ...mapGetters(['totalItems', 'totalPrice'])
  },
  methods: {
    ...mapMutations(['addToCart'])
  }
}
</script>

عند الضغط على الزر يتم تحديث بيانات السلة في store،
ويُحدث العرض تلقائيًا في أي مكون آخر يستخدم نفس الحالة.

export default {
  data() {
    return {
      email: '',
      password: '',
      error: ''

    register() {

        this.error = 'يجب إدخال بريد إلكتروني صحيح'
        return
      }

        this.error = 'كلمة المرور يجب ألا تقل عن 6 أحرف'

      this.error = ''

}
</script>

Vue تستخدم صيغة double curly braces {{ }} لعرض القيم من الكود داخل واجهة المستخدم. 

<div id="app">
  <h1>مرحبا {{ name }}</h1>
</div>

<script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>
<script>
  const { createApp } = Vue
  createApp({
    data() {
      return {
        name: 'معتز محمدي'
      }
    }
  }).mount('#app')
</script>

🧠 شرح:

  • {{ name }} يتم استبدالها بالقيمة الموجودة في data().
  • عندما تتغير القيمة، يتم تحديث النص مباشرة.

🔗 2. الربط مع الخصائص (Attribute Binding)

تستخدم التوجيه v-bind لربط بيانات من الجافاسكريبت مع عناصر HTML.

<div id="app">
  <img v-bind:src="imageUrl" alt="صورة Vue" width="200">
</div>

<script>
  const { createApp } = Vue
  createApp({
    data() {
      return {
        imageUrl: 'https://vuejs.org/images/logo.png'
      }
    }
  }).mount('#app')
</script>

🔹 ملاحظة: يمكنك اختصار v-bind: إلى : فقط مثل:

<img :src="imageUrl">

⚙️ 3. الأحداث (Event Handling)

تُستخدم v-on للاستماع إلى الأحداث مثل النقر، الكتابة، أو التمرير.

<div id="app">
  <p>عدد النقرات: {{ count }}</p>
  <button v-on:click="increment">اضغط هنا</button>
</div>

<script>
  const { createApp } = Vue
  createApp({
    data() {
      return { count: 0 }
    },
    methods: {
      increment() {
        this.count++
      }
    }
  }).mount('#app')
</script>

🪄 اختصار: يمكنك كتابة @click="increment" بدلاً من v-on:click="increment".

🧠 4. الربط الثنائي (Two-Way Binding)

من أقوى ميزات Vue وجود v-model، الذي يربط بين قيمة العنصر والبيانات في الاتجاهين.

<div id="app">
  <input v-model="message" placeholder="اكتب شيئاً">
  <p>النص: {{ message }}</p>
</div>

<script>
  const { createApp } = Vue
  createApp({
    data() {
      return { message: '' }
    }
  }).mount('#app')
</script>

🧩 الآن كلما كتبت في الحقل، يتم تحديث القيمة في الوقت الحقيقي.

🔄 5. الشروط (Conditional Rendering)

Vue توفر توجيهات للتحكم في إظهار العناصر بناءً على شرط.

✅ v-if

<p v-if="isLoggedIn">مرحباً بك، {{ user }}</p>
<p v-else>الرجاء تسجيل الدخول</p>

data() {
  return {
    isLoggedIn: true,
    user: 'معتز'
  }
}

🟡 v-show

يعمل مثل v-if لكن لا يزيل العنصر من DOM، فقط يخفيه بـ display: none.

<p v-show="show">هذا النص يظهر ويختفي</p>
<button @click="show = !show">تبديل</button>

🔁 6. التكرار (Loops)

توجيه v-for يُستخدم لتكرار العناصر بسهولة.

<ul>
  <li v-for="(course, index) in courses" :key="index">
    {{ index + 1 }} - {{ course }}
  </li>
</ul>

<script>
data() {
  return {
    courses: ['HTML', 'CSS', 'JavaScript', 'Vue.js']
  }
}
</script>

🧠 ملاحظة:
key ضرورية لتحسين الأداء وتعريف العناصر بشكل فريد عند التحديث.

🧮 7. الخصائص المحسوبة (Computed Properties)

تُستخدم عندما تحتاج إلى تنفيذ عمليات تعتمد على البيانات، وتُحدّث تلقائياً عند تغييرها.

<div id="app">
  <p>العدد: {{ count }}</p>
  <p>العدد مضروب ×2 = {{ doubleCount }}</p>
  <button @click="count++">+</button>
</div>

<script>
const { createApp } = Vue
createApp({
  data() {
    return { count: 0 }
  },
  computed: {
    doubleCount() {
      return this.count * 2
    }
  }
}).mount('#app')
</script>

🧠 الفرق بين computed و methods:

  • computed تُخزّن النتيجة مؤقتاً (caching).
  • methods يتم تنفيذها في كل مرة يُعاد فيها الرسم (re-render).

🧭 8. المراقبة (Watchers)

تسمح لك watch بمراقبة القيم وتنفيذ أكشن عندما تتغير.

<div id="app">
  <input v-model="name">
  <p>{{ message }}</p>
</div>

<script>
const { createApp } = Vue
createApp({
  data() {
    return {
      name: '',
      message: ''
    }
  },
  watch: {
    name(newValue) {
      this.message = 'أهلاً، ' + newValue
    }
  }
}).mount('#app')
</script>

💡 9. الفلاتر (Filters) – (تم إزالتها في Vue 3)

في Vue 2 كانت هناك "فلاتر" لتنسيق القيم في القوالب، لكن في Vue 3 يُستخدم الآن computed أو methods بدلاً منها.

🧩 1. ما هو المكون في Vue.js؟

المكون هو وحدة مستقلة داخل التطبيق تمثل جزءًا من الواجهة يمكن إعادة استخدامه.
كل مكون يحتوي على:

  • قالب (template) لواجهة المستخدم.
  • منطق (script) للتحكم بالبيانات والسلوك.
  • نمط (style) لتصميمه.

🔹 مثال على مكون Vue (ملف واحد .vue):

<!-- UserCard.vue -->
<template>
  <div class="card">
    <h3>{{ name }}</h3>
    <p>العمر: {{ age }}</p>
  </div>
</template>

<script>
export default {
  props: ['name', 'age']
}
</script>

<style>
.card {
  border: 1px solid #ddd;
  border-radius: 8px;
  padding: 10px;
  margin: 10px;
}
</style>

⚙️ 2. كيفية استخدام المكون

الآن نضيف المكون داخل تطبيقنا الرئيسي.

<!-- App.vue -->
<template>
  <div>
    <h2>قائمة المستخدمين</h2>
    <UserCard name="معتز" age="30" />
    <UserCard name="أحمد" age="25" />
  </div>
</template>

<script>
import UserCard from './components/UserCard.vue'

export default {
  components: { UserCard }
}
</script>

🧠 النتيجة:
سيظهر مكون “بطاقة مستخدم” مكرر مرتين مع بيانات مختلفة — أي أننا أنشأنا مكون قابل لإعادة الاستخدام.

💡 3. تمرير البيانات باستخدام Props

Props هي الطريقة الرسمية لتمرير البيانات من المكون الأب (Parent) إلى المكون الابن (Child).

<!-- ProductCard.vue -->
<template>
  <div>
    <h3>{{ title }}</h3>
    <p>السعر: {{ price }} ر.س</p>
  </div>
</template>

<script>
export default {
  props: {
    title: String,
    price: Number
  }
}
</script>

الاستخدام:

<ProductCard title="هاتف سامسونج" :price="2500" />

🧩 ملاحظة:
إذا كانت القيمة رقمية أو متغير JavaScript، يجب استخدام : قبل اسم الخاصية.

🔁 4. إرسال الأحداث من الابن إلى الأب (Emitting Events)

أحياناً يحتاج المكون الابن إلى إعلام المكون الأب بحدوث حدث (مثل ضغط زر أو تغيير حالة).

<!-- ChildButton.vue -->
<template>
  <button @click="$emit('clicked')">اضغط هنا</button>
</template>

الاستخدام داخل المكون الأب:

<!-- App.vue -->
<template>
  <ChildButton @clicked="handleClick" />
</template>

<script>
import ChildButton from './components/ChildButton.vue'

export default {
  components: { ChildButton },
  methods: {
    handleClick() {
      alert('تم الضغط على الزر من المكون الابن!')
    }
  }
}
</script>

🧠 شرح:

  • $emit('clicked') يرسل حدث مخصص باسم clicked.
  • المكون الأب يستمع له بـ @clicked="handleClick".

🧮 5. التواصل بين المكونات (Communication)

📤 من الأب إلى الابن → باستخدام props

📥 من الابن إلى الأب → باستخدام emit

أما بين مكونات غير مباشرة (أحفاد) فهناك أدوات مثل:

  • Provide / Inject
  • أو استخدام Pinia / Vuex (لإدارة الحالة المركزية).

🎨 6. المكونات الديناميكية (Dynamic Components)

يمكنك تبديل المكون المعروض حسب الحالة بسهولة:

<template>
  <component :is="currentComponent"></component>

  <button @click="toggle">تبديل المكون</button>
</template>

<script>
import LoginForm from './LoginForm.vue'
import RegisterForm from './RegisterForm.vue'

export default {
  data() {
    return { currentComponent: 'LoginForm' }
  },
  components: { LoginForm, RegisterForm },
  methods: {
    toggle() {
      this.currentComponent =
        this.currentComponent === 'LoginForm' ? 'RegisterForm' : 'LoginForm'
    }
  }
}
</script>

💎 7. الـ Slots — تمرير محتوى إلى داخل المكون

الـ slot يسمح للمكون الأب بإرسال محتوى HTML إلى داخل المكون الابن.

<!-- Card.vue -->
<template>
  <div class="box">
    <slot></slot>
  </div>
</template>

الاستخدام:

<Card>
  <h2>عنوان داخل الكارد</h2>
  <p>هذا المحتوى مخصص لهذا المثال.</p>
</Card>

🧠 يمكنك أيضاً استخدام "Named Slots":

<!-- Card.vue -->
<template>
  <div>
    <header><slot name="header"></slot></header>
    <main><slot></slot></main>
    <footer><slot name="footer"></slot></footer>
  </div>
</template>

<Card>
  <template #header><h2>العنوان</h2></template>
  <p>نص داخل المحتوى الرئيسي</p>
  <template #footer><small>© 2025</small></template>
</Card>

⚡ 8. دورة حياة المكون (Lifecycle Hooks)

كل مكون في Vue يمر بمراحل محددة عند إنشائه أو تدميره.

المرحلة الوظيفة

mounted() بعد ظهور المكون على واجهة المستخدم
updated() عند تغيير البيانات وإعادة التحديث
unmounted() عند إزالة المكون 
export default {
  mounted() {
    console.log('تم تحميل المكون!')
  }
}

🚀 1. ما هو Vue Router؟

Vue Router هو المكتبة الرسمية للتنقل بين الصفحات في تطبيقات Vue.js.
وهو الذي يحوّل تطبيقك من صفحة واحدة إلى تطبيق متعدد الصفحات داخل واجهة واحدة فقط.

🔹 أي أنك بدلاً من تحميل صفحة جديدة من السيرفر، يقوم Vue Router بعرض المكون المناسب مباشرة داخل المتصفح بدون إعادة تحميل الصفحة.

import { createRouter, createWebHistory } from 'vue-router'

import AboutView from './views/AboutView.vue'

const routes = [
  { path: '/', component: HomeView },
  { path: '/about', component: AboutView }
]

const router = createRouter({
  history: createWebHistory(),
  routes
})

export default router

🔐 9. حماية الصفحات (Navigation Guards)

تستطيع منع الوصول إلى صفحات معينة إلا بشروط محددة (مثل تسجيل الدخول).

router.beforeEach((to, from, next) => {
  const isAuthenticated = false // افترض أنه لم يسجّل الدخول
  if (to.path === '/profile' && !isAuthenticated) {
    next('/') // ارجعه إلى الصفحة الرئيسية
  } else {
    next()
  }
})

this.$router.push({ name: 'user', params: { id: 10 } })

🟩 الفصل الحادي عشر: نظام التوجيه (Vue Router)

مقدمة

في التطبيقات الحديثة، لا يكفي أن تكون لدينا صفحة واحدة، بل نحتاج إلى الانتقال بين صفحات متعددة (مثل: الصفحة الرئيسية، صفحة المنتجات، صفحة التفاصيل... إلخ) دون إعادة تحميل الصفحة بأكملها.
هنا يأتي دور Vue Router، وهو نظام التوجيه الرسمي لـ Vue.js.

1. ما هو Vue Router؟

هو مكتبة رسمية من Vue.js تُستخدم لإدارة التنقل (navigation) داخل التطبيق أحادي الصفحة (SPA).
يسمح لك بإنشاء روابط بين المكونات المختلفة، وتحديد أي مكون يتم عرضه لكل مسار (Route).

مثال بسيط:

import { createRouter, createWebHistory } from 'vue-router'
import Home from './components/Home.vue'
import About from './components/About.vue'

const routes = [
  { path: '/', component: Home },
  { path: '/about', component: About }
]

const router = createRouter({
  history: createWebHistory(),
  routes
})

export default router

2. تركيب Vue Router

إذا كنت تستخدم Vue CLI:

npm install vue-router

ثم أضف الكود التالي في ملف main.js:

import { createApp } from 'vue'
import App from './App.vue'
import router from './router'

createApp(App).use(router).mount('#app')

3. إنشاء مجلد router

عادةً في تطبيقات Vue الكبيرة، نُنشئ مجلدًا باسم router يحتوي على ملف index.js لإعداد جميع المسارات:

// src/router/index.js
import { createRouter, createWebHistory } from 'vue-router'
import Home from '../views/Home.vue'
import About from '../views/About.vue'

const routes = [
  { path: '/', name: 'Home', component: Home },
  { path: '/about', name: 'About', component: About },
]

const router = createRouter({
  history: createWebHistory(),
  routes
})

export default router

4. استخدام الروابط داخل التطبيق

بدلاً من استخدام <a href=""> نستخدم <router-link> لأنه لا يقوم بإعادة تحميل الصفحة بالكامل:

<template>
  <nav>
    <router-link to="/">الصفحة الرئيسية</router-link> |
    <router-link to="/about">حول</router-link>
  </nav>

  <router-view />
</template>

ما يحدث هنا:

  • <router-link> ينقل المستخدم بين الصفحات دون إعادة تحميل الصفحة.
  • <router-view> هو المكان الذي يتم فيه عرض المكون الحالي المرتبط بالمسار.

5. المسارات الديناميكية (Dynamic Routes)

يمكنك تمرير معلمات في المسار مثل /user/10:

const routes = [
  { path: '/user/:id', component: UserProfile }
]

في داخل المكون UserProfile يمكننا الوصول إلى المعرف كالتالي:

<script>
export default {
  mounted() {
    console.log(this.$route.params.id)
  }
}
</script>

6. التوجيه البرمجي (Programmatic Navigation)

يمكنك التنقل باستخدام كود JavaScript بدلاً من <router-link>:

this.$router.push('/about')

أو باستخدام الاسم:

this.$router.push({ name: 'About' })

7. الحماية من الوصول (Route Guards)

يمكنك حماية صفحات معينة مثل لوحة الإدارة:

router.beforeEach((to, from, next) => {
  const isAuthenticated = false
  if (to.name === 'Admin' && !isAuthenticated) next('/')
  else next()
})

8. التوجيه المتداخل (Nested Routes)

يمكنك إنشاء مسارات بداخل مسارات:

const routes = [
  {
    path: '/users',
    component: Users,
    children: [
      { path: ':id', component: UserDetails }
    ]
  }
]

9. الوضعان: history و hash

  • createWebHistory(): يجعل الروابط جميلة مثل /about (يتطلب إعداد خادم).
  • createWebHashHistory(): يستخدم # مثل /#/about (لا يحتاج إعدادات خادم).

1. ما هي الحالة (State)؟

الحالة هي مصدر الحقيقة (source of truth) في التطبيق،
مثل:

  • المستخدم الحالي
  • سلة المشتريات
  • إعدادات اللغة
  • أو أي بيانات تُستخدم في أكثر من مكون واحد

بدل تمرير هذه البيانات يدويًا بين المكونات، نضعها في Vuex Store.

إذا كنت تستخدم Vue CLI:

3. إعداد المخزن الأساسي (Store)

import { createStore } from 'vuex'

const store = createStore({
  state() {
    return {
      count: 0
    }
  },
  mutations: {
    increment(state) {
      state.count++
    }
  }
})

export default store

ثم في main.js:

import { createApp } from 'vue'
import App from './App.vue'
import store from './store'

createApp(App).use(store).mount('#app')

4. الوصول إلى الحالة في المكونات

<template>
  <div>
    <p>القيمة الحالية: {{ $store.state.count }}</p>
    <button @click="$store.commit('increment')">زيادة</button>
  </div>
</template>

5. مفهوم الـ Mutations و Actions

الجزء الوظيفة
Mutations لتعديل الحالة بشكل مباشر (synchronous)
Actions لتنفيذ عمليات غير متزامنة (مثل جلب البيانات من API)

مثال:

mutations: {
  setUser(state, user) {
    state.user = user
  }
},
actions: {
  async fetchUser({ commit }) {
    const res = await fetch('https://jsonplaceholder.typicode.com/users/1')
    const data = await res.json()
    commit('setUser', data)
  }
}

في المكون:

<template>
  <div>
    <button @click="$store.dispatch('fetchUser')">جلب المستخدم</button>
    <p>{{ $store.state.user?.name }}</p>
  </div>
</template>

6. الـ Getters (المُستخلصات)

تُستخدم للحصول على بيانات مشتقة من الحالة (state).

مثال:

getters: {
  doubleCount(state) {
    return state.count * 2
  }
}

في المكون:

<p>القيمة المضاعفة: {{ $store.getters.doubleCount }}</p>

7. تقسيم المخزن إلى وحدات (Modules)

في المشاريع الكبيرة، يُفضل تقسيم الـ store إلى أجزاء:

const userModule = {
  state: () => ({ name: '', email: '' }),
  mutations: {
    setUser(state, user) {
      state.name = user.name
      state.email = user.email
    }
  }
}

export default createStore({
  modules: {
    user: userModule
  }
})

الاستخدام:

<p>اسم المستخدم: {{ $store.state.user.name }}</p>

8. استيراد المكونات بسهولة عبر الـ Helpers

Vuex يقدم دوال تساعدك في كتابة كود أنظف:

import { mapState, mapActions } from 'vuex'

export default {
  computed: {
    ...mapState(['count'])
  },
  methods: {
    ...mapActions(['fetchUser'])
  }
}

9. Vuex vs Pinia (مستقبل الإدارة في Vue)

1. ما هي Axios؟

Axios هي مكتبة JavaScript تُستخدم لإرسال طلبات HTTP إلى الخادم،
وتُعتبر خيارًا شائعًا وسهل الاستخدام في تطبيقات Vue.

  • تدعم Promises
  • تدعم interceptors (التحكم في الطلبات والردود)

أحد الأسباب التي تجعل Vue.js شائعة جدًا هو نظام المكونات (Components System).
كل جزء من واجهة المستخدم يمكن أن يُمثل مكونًا مستقلاً — مثل زر، أو نموذج، أو بطاقة منتج، إلخ.

1. مفهوم المكونات القابلة لإعادة الاستخدام

المكون القابل لإعادة الاستخدام هو مكون:

  • مستقل في تصميمه ووظيفته
  • يقبل خصائص (Props) لتغيير سلوكه
  • يرسل أحداث (Events) لإعلام المكونات الأخرى بالتغييرات
  • قابل للتخصيص عبر Slots و Props

Mixins تسمح لك بإعادة استخدام الدوال والبيانات بين المكونات.

  }
}

9. المكونات المركبة (Composition Components)

باستخدام Composition API يمكنك إنشاء وظائف قابلة لإعادة الاستخدام بدلاً من Mixins:

// composables/useCounter.js
import { ref } from 'vue'

export function useCounter() {
  const count = ref(0)
  const increment = () => count.value++
  return { count, increment }
}

الاستخدام:

import { useCounter } from '@/composables/useCounter'

export default {
  setup() {
    const { count, increment } = useCounter()
    return { count, increment }
  }
}

10. أفضل الممارسات في تصميم المكونات

1. ما هي الانتقالات في Vue؟

الانتقال (Transition) هو تأثير بصري يحدث عندما يظهر أو يختفي عنصر من الصفحة.
Vue توفر وسمًا جاهزًا لذلك:

<transition>...</transition>

2. مثال بسيط على الانتقال

<template>
  <div>
    <button @click="show = !show">
      تبديل العنصر
    </button>

    <transition name="fade">
      <p v-if="show">مرحبًا بك في Vue.js 🎉</p>
    </transition>
  </div>
</template>

<script>
export default {
  data() {
    return { show: true }
  }
}
</script>

<style>
.fade-enter-active, .fade-leave-active {
  transition: opacity 0.5s;
}
.fade-enter-from, .fade-leave-to {
  opacity: 0;
}
</style>

✅ الشرح:

  • .fade-enter-active: يبدأ عند دخول العنصر.
  • .fade-leave-active: يبدأ عند خروجه.
  • .fade-enter-from و .fade-leave-to: تحدد بداية ونهاية الشفافية.

3. الانتقال باستخدام الحركة (Slide Transition)

<transition name="slide">
  <div v-if="show" class="box"></div>
</transition>

<style>
.slide-enter-active, .slide-leave-active {
  transition: all 0.5s ease;
}
.slide-enter-from {
  transform: translateX(-100px);
  opacity: 0;
}
.slide-leave-to {
  transform: translateX(100px);
  opacity: 0;
}
.box {
  width: 100px;
  height: 100px;
  background: #42b983;
}
</style>

4. استخدام Transition مع القوائم (Transition Group)

عند التعامل مع عناصر متعددة داخل v-for نستخدم:

<transition-group name="list" tag="ul">
  <li v-for="item in items" :key="item" class="list-item">{{ item }}</li>
</transition-group>

.list-enter-active, .list-leave-active {
  transition: all 0.5s;
}
.list-enter-from, .list-leave-to {
  opacity: 0;
  transform: translateY(20px);
}

5. استخدام مكتبات خارجية مثل Animate.css

يمكنك دمج Vue مع مكتبة Animate.css بسهولة:

<transition
  enter-active-class="animate__animated animate__fadeIn"
  leave-active-class="animate__animated animate__fadeOut"
>
  <p v-if="show">نص متحرك باستخدام Animate.css</p>
</transition>

ولا تنسَ إضافة المكتبة في المشروع:

npm install animate.css

ثم في main.js:

import 'animate.css'

6. الانتقالات باستخدام JavaScript Hooks

Vue توفر دوالًا تُنفَّذ أثناء مراحل الانتقال.
مثلاً: عند الدخول أو الخروج.

<transition
  @before-enter="beforeEnter"
  @enter="enter"
  @leave="leave"
>
  <div v-if="show" class="js-box"></div>
</transition>

<script>
export default {
  data() { return { show: true } },
  methods: {
    beforeEnter(el) {
      el.style.opacity = 0
    },
    enter(el, done) {
      setTimeout(() => {
        el.style.transition = 'opacity 1s'
        el.style.opacity = 1
        done()
      }, 100)
    },
    leave(el, done) {
      el.style.transition = 'opacity 1s'
      el.style.opacity = 0
      setTimeout(done, 1000)
    }
  }
}
</script>

7. الانتقال بين الصفحات (Page Transitions)

عند استخدام Vue Router يمكننا تطبيق الانتقالات بين الصفحات.

<template>
  <transition name="fade" mode="out-in">
    <router-view />
  </transition>
</template>

<style>
.fade-enter-active, .fade-leave-active {
  transition: opacity 0.4s;
}
.fade-enter-from, .fade-leave-to {
  opacity: 0;
}
</style>

🔹 mode="out-in" تعني أن الصفحة القديمة تختفي أولًا قبل أن تظهر الجديدة.

8. الانتقالات الشرطية (Conditional Transitions)

يمكنك تغيير نوع الحركة حسب الحالة:

<transition :name="show ? 'fade' : 'slide'">
  <div v-if="show">عنصر متغير التأثير</div>
</transition>

1. ما هي Vue.js؟

Vue.js هي إطار عمل (Framework) أو بالأدق مكتبة تفاعلية (Reactive Library) تُستخدم لبناء واجهات المستخدم (User Interfaces) في تطبيقات الويب.
تم تطويرها بواسطة Evan You في عام 2014، بهدف الجمع بين بساطة مكتبة jQuery وقوة Angular دون التعقيد الزائد.

Vue.js تركّز على طبقة العرض (View Layer) في تطبيق الويب، مما يجعلها خفيفة، سهلة التعلم، وسريعة في الأداء.
يمكنك استخدامها لإنشاء مكون صغير (Component) داخل موقعك، أو لبناء تطبيق متكامل (SPA – Single Page Application).

2. لماذا نستخدم Vue.js؟

في عالم تطوير الويب الحديث، السرعة، التفاعل، وتجربة المستخدم أصبحت عوامل أساسية لنجاح أي تطبيق.
وهنا يأتي دور Vue.js، التي تقدم حلولًا قوية وسهلة لبناء واجهات ديناميكية.

📈 أهم الأسباب لاستخدام Vue.js:

  1. ✅ سهولة التعلم:
    • تعتمد على HTML وCSS وJavaScript التي يعرفها أغلب المطورين.
    • لا تحتاج إلى إعدادات معقدة أو أدوات كثيرة في البداية.
  2. ⚡ أداء عالي:
    • حجمها صغير جدًا (حوالي 20 كيلوبايت بعد الضغط).
    • تستخدم نظام إعادة النشاط (Reactivity System) الفريد الذي يجعل تحديث الواجهة تلقائيًا عندما تتغير البيانات.
  3. 🧩 قائمة على المكونات (Component-Based):
    • تقسّم التطبيق إلى أجزاء مستقلة يمكن إعادة استخدامها بسهولة.
  4. 🔄 ربط البيانات (Two-Way Data Binding):
    • تربط البيانات بين الواجهة (View) والمنطق (Model) بسهولة مثل Angular.
  5. 🛠️ تكامل مرن مع أي مشروع:

6. كيف تعمل Vue.js باختصار؟

عندما تنشئ تطبيق Vue:

  • تكتب البيانات (data) في كائن داخل مكونك.
  • تعرضها في واجهة HTML باستخدام قوالب (templates).
  • وعندما تتغير البيانات، يتم تحديث الواجهة تلقائيًا دون إعادة تحميل الصفحة.

مثال بسيط:

<div id="app">
  <p>{{ message }}</p>
  <button @click="reverseMessage">اعكس النص</button>
</div>

<script>
  const app = Vue.createApp({
    data() {
      return {
        message: 'مرحبًا بك في Vue.js!'
      }
    },
    methods: {
      reverseMessage() {
        this.message = this.message.split('').reverse().join('')
      }
    }
  }).mount('#app')
</script>

📊 النتيجة:

  • النص يظهر على الصفحة.
  • عند الضغط على الزر، يتم عكسه مباشرة دون أي إعادة تحميل.

🗃️ 4. إنشاء المتجر (Pinia Store)

في src/store/tasks.js:

import { defineStore } from 'pinia'

export const useTaskStore = defineStore('taskStore', {
  state: () => ({
    tasks: []
  }),
  actions: {
    addTask(title) {
      if (title.trim() === '') return
      this.tasks.push({
        id: Date.now(),
        title,
        done: false
      })
    },
    removeTask(id) {
      this.tasks = this.tasks.filter(t => t.id !== id)
    },
    toggleDone(id) {
      const task = this.tasks.find(t => t.id === id)
      if (task) task.done = !task.done
    }
  }
})

    localStorage.setItem('tasks', JSON.stringify(this.tasks))
  }
}

🧱 1. مقدمة في أمان واجهات المستخدم

Vue.js تعمل بالكامل في الواجهة الأمامية (Frontend)، أي أن الكود يكون مرئيًا لأي مستخدم.
لذلك من المهم أن تتذكر:

⚠️ لا يمكن الاعتماد على Vue.js وحدها للحماية الكاملة، لأن الأمن الحقيقي يكون في الخادم (Backend).

لكن Vue.js يمكنها المساعدة في:

  • إدارة جلسات الدخول (Sessions) بأمان.
  • حماية البيانات الحساسة من التسرب.
  • منع الهجمات الشائعة مثل XSS وCSRF.

🔐 2. التعامل مع API بطريقة آمنة

عند الاتصال بواجهة API، لا تضع مفاتيح API أو أسرار (Secrets) مباشرة في الكود.

❌ خطأ:

const apiKey = "12345-SECRET";
fetch(`https://api.example.com/data?key=${apiKey}`)

✅ صحيح:

  • خزّن المفاتيح في ملف .env
  • استخدم المتغيرات البيئية (Environment Variables)

في مشروع Vite:
أنشئ ملف .env:

VITE_API_URL=https://api.example.com

واستخدمه داخل الكود:

const api = import.meta.env.VITE_API_URL
fetch(`${api}/tasks`)

🔑 3. المصادقة باستخدام JWT (JSON Web Token)

🔸 ما هو JWT؟

هو رمز (Token) يُصدر من الخادم بعد تسجيل الدخول بنجاح، ويُستخدم للتحقق من هوية المستخدم في الطلبات التالية.

الرمز يكون بهذا الشكل:

eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

🔸 طريقة العمل (Workflow)

  1. المستخدم يُدخل بياناته في نموذج تسجيل الدخول.
  2. Vue ترسلها إلى الخادم (Backend).
  3. الخادم يتحقق من صحة البيانات ويُرجع JWT Token.
  4. Vue تخزن هذا الرمز في LocalStorage أو Cookie آمن.
  5. كل مرة يرسل التطبيق طلبًا جديدًا، يُرسل الرمز في Header: 
    Authorization: Bearer <token>

🧩 مثال عملي في Vue.js

1️⃣ نموذج تسجيل الدخول

<template>
  <form @submit.prevent="login">
    <input v-model="email" type="email" placeholder="البريد الإلكتروني" />
    <input v-model="password" type="password" placeholder="كلمة المرور" />
    <button>تسجيل الدخول</button>
  </form>
</template>

<script setup>
import { ref } from 'vue'

const email = ref('')
const password = ref('')

async function login() {
  const res = await fetch('https://api.example.com/login', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ email: email.value, password: password.value })
  })

  const data = await res.json()
  localStorage.setItem('token', data.token)
}
</script>

2️⃣ استخدام الرمز في الطلبات اللاحقة

async function getTasks() {
  const token = localStorage.getItem('token')
  const res = await fetch('https://api.example.com/tasks', {
    headers: {
      Authorization: `Bearer ${token}`
    }
  })
  return await res.json()
}

🧠 4. حماية الواجهات (Route Guards)

نستخدم Vue Router لمنع المستخدمين من الوصول إلى صفحات محمية بدون تسجيل الدخول.

في router/index.js:

import { createRouter, createWebHistory } from 'vue-router'
import Home from '../views/HomeView.vue'
import Login from '../views/LoginView.vue'

const routes = [
  { path: '/', component: Home, meta: { requiresAuth: true } },
  { path: '/login', component: Login }
]

const router = createRouter({
  history: createWebHistory(),
  routes
})

// حماية الصفحات
router.beforeEach((to, from, next) => {
  const token = localStorage.getItem('token')
  if (to.meta.requiresAuth && !token) {
    next('/login')
  } else {
    next()
  }
})

export default router

لن يتمكن أي مستخدم غير مسجل الدخول من الوصول إلى الصفحة الرئيسية.

🧩 6. منع الهجمات الشائعة

🛡️ أ. XSS (Cross Site Scripting)

يحصل عندما يتم إدخال كود JavaScript داخل حقول الإدخال.

مثال خطر:

<input value="<script>alert('Hacked')</script>">

✅ الحل في Vue:

  • Vue تُعقّم (sanitize) القيم تلقائيًا.
  • لا تستخدم v-html إلا عند الضرورة، وتأكد من تنظيف البيانات قبل عرضها.

🔁 ب. CSRF (Cross Site Request Forgery)

الهجوم الذي يرسل طلبات مزيفة باستخدام Cookie المستخدم.

✅ الحل:

  • لا تعتمد على Cookies وحدها.
  • استخدم Tokens في Header بدلاً من الاعتماد على Session Cookies.

🔒 ج. حماية الروابط والإدخالات

  • تحقق دائمًا من صحة البيانات قبل إرسالها.
  • لا تعرض معلومات حساسة في واجهة المستخدم.
  • استخدم HTTPS دائمًا.

export const useTaskStore = defineStore("taskStore", {
  state: () => ({
    tasks: JSON.parse(localStorage.getItem("tasks") || "[]"),
  }),
  actions: {
    addTask(title) {
      const newTask = { id: Date.now(), title, completed: false };
      this.tasks.push(newTask);
      this.saveTasks();
    },
    deleteTask(id) {
      this.tasks = this.tasks.filter((t) => t.id !== id);
      this.saveTasks();
    },
    toggleTask(id) {
      const task = this.tasks.find((t) => t.id === id);
      if (task) task.completed = !task.completed;
      this.saveTasks();
    },
    saveTasks() {
      localStorage.setItem("tasks", JSON.stringify(this.tasks));
    },
  },
});

  name: "App",
  components: { TaskForm, TaskList },
  setup() {
    const taskStore = useTaskStore();
    const { tasks } = storeToRefs(taskStore);

    return {
      tasks,
      addTask: taskStore.addTask,
      deleteTask: taskStore.deleteTask,
    };
  },
};
</script>
