يا فندم، الفصل الرابع ده هو بداية الشغل الحقيقي والعمليات الأساسية في الـ MySQL. بما إننا عرفنا في الفصل اللي فات إزاي ندخل أوامر (Queries)، الفصل ده بيعلمنا إزاي نعمل قاعدة بيانات وجداول ونحط فيها البيانات ونسترجعها بألف طريقة وطريقة.

المرجع بيستخدم مثال اسمه menagerie (يعني حديقة حيوانات أليفة) عشان يسجل معلومات عن حيواناتك.

ده شرح مفصل للفصل الرابع: إنشاء واستخدام قاعدة بيانات (Creating and Using a Database).

4.1 إنشاء واختيار قاعدة بيانات (Creating and Selecting a Database)

الخطوة الأولى هي إننا نعمل مكان للبيانات دي (قاعدة البيانات).

  1. رؤية قواعد البيانات الموجودة: قبل ما تبدأ، ممكن تشوف إيه القواعد الموجودة بالفعل على السيرفر باستخدام أمر SHOW DATABASES;. 
    mysql> SHOW DATABASES;
+----------+
| Database |
+----------+
| mysql    |
| test     |
| tmp      |
+----------+
  2. إنشاء قاعدة بيانات جديدة: عشان تعمل قاعدة البيانات بتاعتك اللي اسمها menagerie، بتستخدم أمر CREATE DATABASE: 
    mysql> CREATE DATABASE menagerie;
    (ملاحظة: تحت نظام تشغيل زي يونكس/لينكس، أسماء قواعد البيانات حساسة لحالة الحروف (case-sensitive)، عشان كده لازم دايماً تستخدم نفس طريقة الكتابة menagerie، مش MENAGERIE مثلاً). ولو واجهتك مشكلة هنا، ممكن تكون محتاج صلاحيات من المدير.
  3. اختيار قاعدة البيانات للاستخدام: مجرد إنك عملت قاعدة البيانات مش معناها إنك بدأت تشتغل عليها. لازم تحددها صراحة عشان كل أوامرك اللي جاية تتطبق جواها. بنستخدم أمر USE: 
    mysql> USE menagerie;
Database changed
    زي الأمر QUIT، الأمر USE مش بيحتاج فاصلة منقوطة (;) في آخره، وميزة كمان إن لازم يتكتب في سطر واحد. (ممكن كمان تختار قاعدة البيانات دي وأنت بتفتح الـ mysql من الـ command line أصلاً).

4.2 إنشاء جدول (Creating a Table)

بعد ما اخترنا قاعدة البيانات، هي لسه فاضية. لو عملت SHOW TABLES; هتلاقيها فاضية:

mysql> SHOW TABLES;
Empty set (0.00 sec)

دلوقتي محتاجين نعمل الجدول اللي هيشيل بيانات الحيوانات. الجدول ده بنسميه pet.

إيه الأعمدة اللي محتاجينها؟ المرجع بيقول إننا محتاجين نجمع معلومات زي الاسم، والمالك، والنوع (Species)، والجنس (Sex)، وتاريخ الميلاد، وتاريخ الوفاة.

ملحوظة مهمة بخصوص العمر: بيقولك بلاش تخزن العمر في قاعدة البيانات مباشرة!. ليه؟ عشان العمر بيتغير كل يوم، وده معناه إنك هتحتاج تحدث البيانات باستمرار. الأفضل إنك تخزن تاريخ الميلاد (birth date)، ولما تحتاج العمر، بتحسبه عن طريق دوال التاريخ (Date arithmetic). تخزين تاريخ الميلاد ده بيدي مميزات تانية، زي إنك تبعت تهنئة بأعياد الميلاد أو تحسب عمر الحيوان لما مات.

أمر إنشاء الجدول:

بنستخدم أمر CREATE TABLE وبنحدد اسم كل عمود ونوعه:

mysql> CREATE TABLE pet (name VARCHAR(20), owner VARCHAR(20),
        species VARCHAR(20), sex CHAR(1), birth DATE, death DATE);
  • VARCHAR(20): مناسب لـ (name, owner, species) لأن قيمته بتتغير في الطول. الـ 20 دي هي أقصى عدد حروف ممكن تحطه.
  • CHAR(1): مناسب لـ (sex) لو هتستخدم حرف واحد ('f' أو 'm').
  • DATE: مناسب لـ (birth, death).

التأكد من هيكل الجدول: عشان تتأكد إن الجدول اتعمل صح، بتستخدم أمر DESCRIBE (أو اختصارها DESC):

mysql> DESCRIBE pet;
+---------+-------------+------+-----+---------+-------+
| Field   | Type        | Null | Key | Default | Extra |
+---------+-------------+------+-----+---------+-------+
| name    | varchar(20) | YES  |     | NULL    |       |
| owner   | varchar(20) | YES  |     | NULL    |       |
| species | varchar(20) | YES  |     | NULL    |       |
| sex     | char(1)     | YES  |     | NULL    |       |
| birth   | date        | YES  |     | NULL    |       |
| death   | date        | YES  |     | NULL    |       |
+---------+-------------+------+-----+---------+-------+

4.3 إدخال البيانات في جدول (Loading Data into a Table)

بعد ما عملنا الجدول، محتاجين نملأه بالبيانات (Populate). فيه طريقتين مشهورين:

1. استخدام LOAD DATA (لإدخال بيانات كتير مرة واحدة)

لو عندك بيانات كتير (مثلاً سجلات 8 حيوانات)، ممكن تحطها في ملف نصي (زي pet.txt) وتستخدم أمر LOAD DATA.

  • صيغة التواريخ: لازم التواريخ تكون بصيغة 'YYYY-MM-DD' (زي 1993-02-04).
  • القيم الناقصة (Missing Values): لو فيه قيمة مش عارفها (زي جنس Whistler أو تاريخ وفاة حيوان لسه عايش)، بتمثلها في الملف النصي بـ \N (باك سلاش و N كابيتال).
# شكل السطر في ملف pet.txt (بفرض إنهم مفصولين بـ Tab)
Whistler    Gwen    bird    \N      1997-12-09      \N

الأمر اللي بيحمل الملف:

mysql> LOAD DATA LOCAL INFILE '/path/pet.txt' INTO TABLE pet;

(لو الملف معمول على ويندوز، قد تحتاج تضيف LINES TERMINATED BY '\r\n' عشان يتعرف على نهاية السطر صح).

2. استخدام INSERT (لإدخال سجل واحد)

لو عايز تضيف سجل جديد واحد بس (زي حيوان "Puffball" الجديد)، بتستخدم INSERT: 

mysql> INSERT INTO pet
       VALUES ('Puffball','Diane','hamster','f','1999-03-30',NULL);

مقارنة مهمة: في INSERT، لو فيه قيمة مفقودة، بتكتب NULL مباشرة. أما في ملف LOAD DATA كنا بنستخدم \N.

4.4 استرجاع البيانات من جدول (Retrieving Information from a Table)

هنا بقى بنستخدم الـ SELECT عشان نطلع المعلومات اللي محتاجينها.

الصيغة العامة لأمر SELECT:

SELECT اللي_عايز_تشوفه FROM الجدول_اللي_هتجيب_منه WHERE شروط_التحقيق;

4.4.1 اختيار كل البيانات (Selecting All Data)

عشان تشوف الجدول كله:

mysql> SELECT * FROM pet;
  • الـ * دي اختصار لـ "كل الأعمدة".
  • النتيجة هتكون فيها كل السجلات اللي أدخلتها.

تصحيح الأخطاء (التعديل): لو لقيت إن فيه غلط في البيانات (زي إن تاريخ ميلاد Bowser كان المفروض 1989 مش 1979)، ممكن تصلحه باستخدام UPDATE:

mysql> UPDATE pet SET birth = '1989-08-31' WHERE name = 'Bowser';

الـ UPDATE ده بيغير السجل اللي فيه المشكلة بس، ومش محتاج تعيد تحميل الجدول كله.

4.4.2 اختيار صفوف معينة (Selecting Particular Rows)

لو عايز تجاوب على سؤال محدد، بتستخدم جملة WHERE عشان تحدد الشروط:

  • مثال (سجل حيوان معين): عايز أشوف بيانات "Bowser" بس:

    mysql> SELECT * FROM pet WHERE name = 'Bowser';

    (المقارنة بين النصوص هنا مش حساسة لحالة الحروف (case-insensitive)، فممكن تكتب 'bowser' أو 'BOWSER' عادي).

  • مثال (شروط على التاريخ): الحيوانات اللي اتولدت في أو بعد سنة 1998:

    mysql> SELECT * FROM pet WHERE birth >= '1998-1-1';

  • مثال (دمج الشروط - AND): عايزة الكلاب (dog) اللي جنسها أنثى (f):

    mysql> SELECT * FROM pet WHERE species = 'dog' AND sex = 'f';

  • مثال (دمج الشروط - OR): عايزة الثعابين (snake) أو الطيور (bird):

    mysql> SELECT * FROM pet WHERE species = 'snake' OR species = 'bird';

  • ملاحظة الأقواس: لو هتستخدم AND و OR مع بعض، الأفضل تستخدم أقواس عشان تحدد الأولويات، لأن AND ليها أولوية أعلى من OR.

4.4.3 اختيار أعمدة معينة (Selecting Particular Columns)

عشان تشوف معلومات معينة بس، بتكتب أسماء الأعمدة بدل *:

  • مثال: عايزة الاسم وتاريخ الميلاد بس:

    mysql> SELECT name, birth FROM pet;

  • مثال (إزالة التكرار): عشان تعرف مين هم الملاك اللي عندهم حيوانات، وعايز كل اسم يظهر مرة واحدة بس:

    mysql> SELECT DISTINCT owner FROM pet;

4.4.4 ترتيب الصفوف (Sorting Rows)

عشان تخلي النتائج تظهر بشكل منظم ومفهوم، بتستخدم جملة ORDER BY:

  • مثال (ترتيب حسب الميلاد تصاعدي): أقدم حيوان أولاً:

    mysql> SELECT name, birth FROM pet ORDER BY birth;

  • مثال (ترتيب حسب الميلاد تنازلي): أحدث حيوان أولاً: بتضيف كلمة DESC (Descending):

    mysql> SELECT name, birth FROM pet ORDER BY birth DESC;

  • مثال (الترتيب بأكثر من عمود): الترتيب حسب النوع (تصاعدي)، وبعدين داخل كل نوع، الترتيب حسب تاريخ الميلاد (تنازلي):

    mysql> SELECT name, species, birth FROM pet
       ORDER BY species, birth DESC;

    (كلمة DESC بتطبق بس على العمود اللي قبلها مباشرة، وهو birth).

4.4.5 حسابات التواريخ (Date Calculations)

MySQL عنده دوال كتير عشان تحسب بيها تواريخ:

  • حساب العمر: بنستخدم الدالة TIMESTAMPDIFF(YEAR, date1, date2). 

    mysql> SELECT name, birth, CURDATE(),
       TIMESTAMPDIFF(YEAR,birth,CURDATE()) AS age
       FROM pet ORDER BY age;

    (استخدمنا AS age عشان نخلي اسم العمود في النتيجة شكله أحسن).

  • حساب العمر عند الوفاة: بنشوف الحيوانات اللي تاريخ وفاتها مش NULL، وبنحسب الفرق:

    mysql> SELECT name, birth, death,
       TIMESTAMPDIFF(YEAR,birth,death) AS age
       FROM pet WHERE death IS NOT NULL ORDER BY age;

  • إيجاد أعياد الميلاد الشهر الجاي: بنستخدم دالة MONTH() اللي بتجيب رقم الشهر، وبنستخدم دوال زي DATE_ADD() عشان نعرف الشهر اللي جاي:

    mysql> SELECT name, birth FROM pet
       WHERE MONTH(birth) = MONTH(DATE_ADD(CURDATE(),INTERVAL 1 MONTH));

    (ممكن نستخدم دالة MOD عشان لو الشهر الحالي ديسمبر (12) يبقى الشهر اللي جاي يناير (1)).

4.4.6 التعامل مع قيم NULL (Working with NULL Values)

الـ NULL معناها "قيمة مجهولة أو مفقودة". وده بيخليها مختلفة شوية عن أي قيمة تانية.

  • القاعدة الذهبية: ممنوع تستخدم علامات المقارنة العادية (=، <، <>) عشان تختبر الـ NULL.
  • لازم تستخدم IS NULL أو IS NOT NULL.
    • عشان كده في مثال حساب عمر الحيوانات اللي ماتت، استخدمنا death IS NOT NULL بدل death <> NULL.
  • نتيجة مقارنة أي شيء بـ NULL هي NULL أيضاً. 
    mysql> SELECT 1 = NULL;
+----------+
| 1 = NULL |
+----------+
|     NULL |
+----------+
  • ملاحظة: الصفر (0) والسلسلة الفارغة ('') ليستا NULL.

4.4.7 مطابقة الأنماط (Pattern Matching)

عشان تبحث عن نصوص معينة (مش لازم تكون متطابقة بالكامل)، عندنا طريقتين:

1. مطابقة أنماط SQL (LIKE / NOT LIKE):

  • %: يمثل أي عدد من الحروف (صفر أو أكثر).

  • _: يمثل حرف واحد بس.

  • مثال (الأسماء اللي تبدأ بـ b):

    mysql> SELECT * FROM pet WHERE name LIKE 'b%';
  • مثال (الأسماء اللي تنتهي بـ fy): 
    mysql> SELECT * FROM pet WHERE name LIKE '%fy';
  • مثال (الأسماء اللي طولها خمس حروف بالظبط): 
    mysql> SELECT * FROM pet WHERE name LIKE '_____';

2. التعبيرات القياسية الموسعة (REGEXP / RLIKE):

دي بتستخدم رموز زي اللي بتشوفها في برامج يونكس (grep).

  • ^: بداية القيمة (النص).
  • $: نهاية القيمة.
  • .: يطابق أي حرف واحد.

  • ميزة REGEXP إن النمط لو اتطابق في أي مكان في القيمة، الـ Query بينجح.

  • مثال (الأسماء اللي تبدأ بـ b):

    mysql> SELECT * FROM pet WHERE name REGEXP '^b';
  • مثال (الأسماء اللي طولها خمس حروف بالظبط): (باستخدام 5 نقاط بين البداية والنهاية): 
    mysql> SELECT * FROM pet WHERE name REGEXP '^.....$';

4.4.8 عد الصفوف (Counting Rows)

عشان نجاوب على أسئلة "كام مرة" أو "ما هو العدد الكلي؟"، بنستخدم دالة COUNT().

  • مثال (العدد الكلي للحيوانات):

    mysql> SELECT COUNT(*) FROM pet;
+----------+
| COUNT(*) |
+----------+
|        9 |
+----------+

  • مثال (العد حسب المجموعات): عشان نعرف كل مالك عنده كام حيوان، بنستخدم GROUP BY:

    mysql> SELECT owner, COUNT(*) FROM pet GROUP BY owner;
+--------+----------+
| owner  | COUNT(*) |
+--------+----------+
| Benny  |        2 |
| Diane  |        2 |
| Gwen   |        3 |
| Harold |        2 |
+--------+----------+

    (بنجمع كل السجلات اللي تخص مالك واحد في مجموعة واحدة، وبعدين بنعد المجموعة دي).

  • مثال (العد حسب النوع والجنس): 
    mysql> SELECT species, sex, COUNT(*) FROM pet GROUP BY species, sex;
    (ملاحظة: لو العمود sex فيه NULL، هيظهر صف منفصل في نتائج العد بيقولك فيه كام سجل جنسه غير معروف).

4.4.9 استخدام أكثر من جدول (Using More Than one Table)

أحياناً، عشان تجاوب على سؤال معين، بتحتاج معلومات متوزعة على أكتر من جدول.

  • الجدول الثاني: عملنا جدول تاني اسمه event عشان نسجل الأحداث (زي الولادة أو زيارة الدكتور):

    mysql> CREATE TABLE event (name VARCHAR(20), date DATE,
       type VARCHAR(15), remark VARCHAR(255));

    (وبعد كده بنملأه بالبيانات زي ما عملنا في جدول pet).

  • مثال (الـ JOIN): لو عايز تعرف عمر الحيوان لما ولَّد (لتر).

    1. تاريخ الولادة موجود في جدول event.
    2. تاريخ ميلاد الأم موجود في جدول pet.
    3. محتاجين نربط الجدولين ببعض عن طريق الاسم.

  • استخدام INNER JOIN:

    mysql> SELECT pet.name,
       TIMESTAMPDIFF(YEAR,birth,date) AS age,
       remark
       FROM pet INNER JOIN event
       ON pet.name = event.name
       WHERE event.type = 'litter';

    إيه اللي حصل هنا؟

    1. الـ FROM pet INNER JOIN event بيحدد الجدولين اللي هنستخدمهم.
    2. الـ ON pet.name = event.name ده هو شرط الربط، بيشوف السجل في pet اللي اسمه متطابق مع السجل في event.
    3. الـ INNER JOIN بيجيب الصفوف اللي ليها تطابق في الجدولين بس.
    4. لما تشوف عمود زي name موجود في الجدولين، لازم تحدد اسم الجدول قبله (زي pet.name و event.name).

  • الـ Self Join: ممكن تربط الجدول بنفسه عشان تقارن السجلات اللي جواه. لو عايز تشوف أزواج التزاوج المحتملة (أنثى حية وذكر حي من نفس النوع)، بتستخدم أسماء مستعارة (Aliases) زي p1 و p2 عشان تمثل الجدول مرتين.

mysql> SELECT p1.name, p1.sex, p2.name, p2.sex, p1.species
       FROM pet AS p1 INNER JOIN pet AS p2
       ON p1.species = p2.species
         AND p1.sex = 'f' AND p1.death IS NULL
         AND p2.sex = 'm' AND p2.death IS NULL;
