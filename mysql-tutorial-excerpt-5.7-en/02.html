إليك الشرح بالتفصيل والأمثلة، يا معلم:

الفصل التاني: الاتصال والانفصال عن السيرفر (Connecting to and Disconnecting from the Server)

الفصل ده بيدور كله حوالين إزاي تفتح الباب وتدخل على سيرفر MySQL عشان تبدأ تكتب الأوامر.

1. متطلبات الاتصال (المفتاح السري والعنوان)

عشان تعرف تتصل بالسيرفر، غالباً هتحتاج توفر ثلاث حاجات أساسية:

  1. اسم مستخدم (MySQL user name): اللي هو اسم الدخول بتاعك.
  2. كلمة مرور (Password): غالباً بتكون مطلوبة.
  3. اسم المضيف (Host Name): لو السيرفر شغال على جهاز تاني غير اللي أنت قاعد عليه، لازم تحدد اسم الجهاز ده.

نصيحة المسؤول: الكتاب بينصح إنك تتكلم مع المسؤول (Administrator) بتاع MySQL عشان تعرف بالظبط إيه هي المعاملات اللي المفروض تستخدمها (زي الـ Host والـ User والـ Password).

2. طريقة الاتصال العامة (اللوج إن)

لما تكون عارف بياناتك، بتستخدم الأمر التالي عشان تتصل بالسيرفر، وبيطلب منك الباسورد بعد كده:

الأمر:

$> mysql -h host -u user -p
Enter password: ********
  • host: ده بتمثل اسم الجهاز أو السيرفر اللي عليه MySQL.
  • user: ده بتمثل اسم المستخدم بتاعك على MySQL.
  • -p: الاختصار ده بيطلب من برنامج mysql إنه يسألك على كلمة المرور، وده أحسن عشان متبانش الباسورد في سطر الأوامر (Command Line).

3. لما الاتصال ينجح

لو الاتصال مشي تمام، هتلاقي البرنامج عرضلك شوية معلومات ترحيبية، وبعدها هيظهر لك البرومبت (prompt) اللي مستني منك الأوامر:

شكل الإخراج بعد النجاح:

Welcome to the MySQL monitor.  Commands end with ; or \g.
Your MySQL connection id is 25338 to server version: 5.7.44-standard
Type 'help;' or '\h' for help.
Type '\c' to clear the buffer.
mysql>

البرومبت ده اللي هو mysql> بيعرفك إنك دلوقتي متصل، وجاهز تكتب أوامر SQL وتنفذها.

4. حالة الاتصال المحلي

لو سيرفر MySQL شغال على نفس الجهاز اللي أنت بتستخدمه عشان تدخل، ممكن تستغنى عن تحديد الـ Host (يعني تشيل -h host) وتستخدم الأمر ده بس:

الأمر للاتصال المحلي:

$> mysql -u user -p

وفي بعض التسطيبات (Installations) لـ MySQL، بيسمحوا للمستخدمين إنهم يدخلوا باسم مستخدم مجهول (anonymous user) لو بيتصلوا من نفس الجهاز، وفي الحالة دي ممكن تدخل ومتحطش أي خيارات خالص:

$> mysql

5. التعامل مع الأخطاء الشائعة

لو حاولت تعمل Log In وجاتلك رسالة خطأ زي دي:

ERROR 2002 (HY000): Can't connect to local MySQL server through socket '/tmp/mysql.sock' (2)

ده معناه إن سيرفر MySQL نفسه مش شغال، سواء كان كخدمة (Service) على ويندوز أو كخادم (Daemon) على يونكس/لينكس. ولازم ترجع للمسؤول عشان يشغله.

6. إزاي تفصل الاتصال (Log Out)

بعد ما تخلص شغلك، ممكن تفصل الاتصال في أي وقت عن طريق كتابة الأمر QUIT أو الاختصار بتاعه \q عند البرومبت بتاع mysql>:

مثال الفصل:

mysql> QUIT
Bye

كمان ممكن تفصل الاتصال على أنظمة يونكس (Unix) بالضغط على Control+D.

ملخص الفصل ده زي...

تخيل إن سيرفر MySQL ده زي خزنة بنك معلومات ضخمة. الفصل الأول عرفك على الأداة اللي هتستخدمها عشان تفتح الخزنة (برنامج mysql). الفصل التاني ده بقى هو اللي بيدي الأوامر الفعلية لفتح باب الخزنة عن طريق إنك بتقدم اسمك وعنوانك (User و Host) وتدخل مفتاحك (Password) عشان يقولك "أهلاً بك" وتبدأ شغل (يظهر لك mysql>). ولما تخلص، بتدوس على زرار "خروج" (QUIT) عشان الباب يتقفل بأمان.
