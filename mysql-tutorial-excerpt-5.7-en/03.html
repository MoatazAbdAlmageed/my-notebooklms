يا فندم، تحت أمرك. بما إنك طالب شرح مفصل باللهجة المصرية وبالأمثلة للفصل رقم 3، اللي بيتكلم عن "إدخال الاستعلامات" (Entering Queries) في دليل MySQL، هنجاوب بالتفصيل الممل عشان تبقى فاهم كل خطوة.

الفصل ده هو الخطوة اللي بتيجي بعد ما تكون عملت اتصال بالسيرفر (زي ما شرح الفصل اللي قبله).

الفصل 3: إدخال الاستعلامات (Entering Queries)

الخلاصة: الفصل ده بيعلمك إزاي تبعت أوامر (Queries) لـ MySQL وإزاي تفهم رد السيرفر عليك، وإيه العلامات اللي بتظهرلك في الشاشة عشان تعرف إنت بتعمل إيه بالظبط.

1. أساسيات كتابة الاستعلام (الـ Query)

لما بتكون فاتح الـ mysql> (الـ monitor أو الـ client)، السيرفر بيكون مستني منك أوامر SQL.

  • القاعدة الأساسية: أي استعلام (Query) أو أمر SQL بتكتبه لازم يخلص بـ فاصلة منقوطة (Semicolon ;).

    • ملحوظة: فيه أوامر استثناء زي أمر الخروج QUIT أو أمر اختيار قاعدة البيانات USE مش بيحتاجوا فاصلة منقوطة، بس لو كتبتها مش هتضر.

  • تنفيذ الأمر: أول ما بتكتب الأمر والفاصلة المنقوطة وتدوس Enter، برنامج mysql بيبعت الأمر للسيرفر عشان ينفذه ويعرض النتيجة، وبعدها بيظهرلك برومبت mysql> تاني عشان تستعد للأمر اللي بعده.

  • الكلمات المفتاحية (Keywords): ممكن تكتب الكلمات المفتاحية بتاعت SQL بأي طريقة تعجبك (كابيتال، سمول، ميكس).

    • مثال: الأوامر دي كلها زي بعض بالظبط: 
      mysql> SELECT VERSION(), CURRENT_DATE;
mysql> select version(), current_date;
mysql> SeLeCt vErSiOn(), current_DATE;

2. أمثلة بسيطة للتجربة

عشان تجرب وتتعود على طريقة شغل الـ mysql، ممكن تجرب الأوامر دي:

أ. معرفة إصدار السيرفر والتاريخ الحالي: الاستعلام ده بسيط وبيجيبلك رقم إصدار السيرفر والتاريخ:

mysql> SELECT VERSION(), CURRENT_DATE;
+--------------+--------------+
| VERSION()    | CURRENT_DATE |
+--------------+--------------+
| 5.7.1-m4-log | 2012-12-25   |
+--------------+--------------+
1 row in set (0.01 sec)

إيه اللي نفهمه من النتيجة؟

  1. الجدول: بيعرض النتيجة في شكل جدول (صفوف وأعمدة).
  2. رؤوس الأعمدة: أسماء الأعمدة بتكون عادةً هي أسماء الحقول اللي بتجيبها من الجدول، أو لو بتجيب قيمة تعبير (Expression) زي المثال ده، بيسمي العمود باسم التعبير نفسه (زي VERSION()).
  3. وقت التنفيذ: بيوريك كام صف رجع (1 row in set) والوقت اللي أخده الأمر عشان يتنفذ (0.01 sec). ده بيديلك فكرة تقريبية عن أداء السيرفر.

ب. استخدام MySQL كآلة حاسبة: الـ mysql ممكن تستخدمه كآلة حاسبة بسيطة:

mysql> SELECT SIN(PI()/4), (4+1)*5;
+------------------+---------+
| SIN(PI()/4)      | (4+1)*5 |
+------------------+---------+
| 0.70710678118655 |      25 |
+------------------+---------+
1 row in set (0.02 sec)

ج. كتابة أكثر من أمر في سطر واحد: ممكن تكتب كذا أمر ورا بعض في سطر واحد، المهم إن كل واحد يخلص بفاصلة منقوطة:

mysql> SELECT VERSION(); SELECT NOW();

3. التعامل مع الاستعلامات اللي على كذا سطر

لو عندك استعلام طويل، مش لازم تكتبه كله في سطر واحد. الـ mysql بيستنى وبيجمع كل السطور اللي بتكتبها ومش بينفذ الأمر إلا لما يشوف الـ فاصلة منقوطة (;).

أ. تغيير علامة الإدخال (الـ Prompt): لما تبدأ تكتب الأمر وتدوس Enter قبل ما تحط الفاصلة المنقوطة، الـ Prompt بيتغير من mysql> إلى ->.

  • مثال للاستعلام متعدد السطور: 
    mysql> SELECT
    -> USER()
    -> ,
    -> CURRENT_DATE;
+---------------+--------------+
| USER()        | CURRENT_DATE |
+---------------+--------------+
| jon@localhost | 2010-08-06   |
+---------------+--------------+
    الـ -> دي هي صديقك! بتوريك إنك لسه بتكتب في استعلام غير مكتمل والسيرفر مستني بقية الأمر.

ب. إلغاء استعلام (Cancellation): لو بدأت تكتب استعلام على كذا سطر وغيرت رأيك ومش عايز تكمله، ممكن تلغيه بإنك تكتب \c. 

mysql> SELECT
    -> USER()
    -> \c
mysql>

بمجرد ما تكتب \c، الـ Prompt بيرجع تاني mysql> وبيقولك إنه جاهز لاستعلام جديد.

4. إشارات الـ Prompt والوقوع في الفخ!

الـ Prompts دي مهمة جداً عشان تعرف الـ mysql مستني منك إيه بالظبط:

الـ Prompt المعنى (بالعامية المصرية)
mysql> جاهز لأمر جديد.
-> مستني بقية الأمر اللي على كذا سطر.
'> مستني تكملة نص (string) بدأته بعلامة اقتباس مفردة (').
"> مستني تكملة نص (string) بدأته بعلامة اقتباس مزدوجة (").
` >` مستني تكملة اسم مُعَرِّف (identifier) بدأته بـ Backtick (` ).
/*> مستني تكملة تعليق (comment) بدأ بـ /*.

فخ النسيان (الفاصلة المنقوطة): أكتر سبب بيخلي الـ Prompt يتحول لـ -> وأنت فاكر إنك خلصت، هو إنك نسيت الـ فاصلة منقوطة (;) بعد الأمر. 

mysql> SELECT USER()
    -> ;  <- مجرد ما تضيف الفاصلة المنقوطة، الأمر بيتنفذ

فخ النصوص (Strings): لو شفت الـ Prompt اتحول لـ '> أو "> ده معناه إنك نسيت علامة الإغلاق للنص.

  • الـ mysql بيسمح إن النصوص تمتد على كذا سطر، ولما بتنسى علامة الإغلاق، بيفضل مستني بقية النص.

  • مثال لخطأ في نص: شوف المثال ده، النقطة الحمراء دي بتقولك إنك نسيت علامة الاقتباس الثانية:

    mysql> SELECT * FROM my_table WHERE name = 'Smith AND age < 30;
    '>

    الـ '> دي هي الدليل بتاعك! هي اللي بتقولك إن الـ Query "نايم" ومستني علامة اقتباس.

طريقة الخروج من فخ النص: لو وقعت في فخ الـ '> أو ">، ما ينفعش تكتب \c لوحدها، لأن الـ mysql هيعتبرها جزء من النص اللي بيجمعه. الحل هو: تكمل النص الناقص (تحط علامة الاقتباس المناسبة)، وبعدها تلغي الأمر. 

    '> '\c
mysql>

الـ Prompt هيرجع تاني mysql>.

فهمك للـ Prompts دي مهم جداً، عشان لو كتبت أمر غلط، مش هتفضل مستني الـ Query اللي مش هيتنفذ، وهتعرف إيه اللي السيرفر مستنيه منك.
