يا فندم، الفصل الخامس ده يعتبر "بطاقة الهوية" بتاعة الشغل بتاعك في MySQL. لو نسيت اسم قاعدة بيانات، أو نسيت إيه الأعمدة اللي في جدول معين، الفصل ده بيوريك إزاي تسترجع المعلومات دي بالظبط.

الفصل الخامس اسمه "الحصول على معلومات حول قواعد البيانات والجداول" (Getting Information About Databases and Tables).

الخلاصة: الفصل ده بيعلمك تستخدم مجموعة أوامر اسمها SHOW و DESCRIBE عشان "تتجسس" على السيرفر وتعرف كل حاجة عن الهيكل اللي بنيته.

1. معرفة قاعدة البيانات الحالية (أنت فين دلوقتي؟)

لو نسيت إنت شغال على أي قاعدة بيانات بالظبط، ممكن تسأل السيرفر مباشرة باستخدام الدالة DATABASE() مع أمر SELECT.

المثال:

mysql> SELECT DATABASE();
+------------+
| DATABASE() |
+------------+
| menagerie  |
+------------+
  • الشرح: لو كنت عامل USE menagerie; قبل كده، السيرفر هيرجع اسم قاعدة البيانات اللي شغال عليها حالياً، اللي هي menagerie.
  • ملحوظة: لو مكنتش مختار أي قاعدة بيانات أصلاً، النتيجة هتكون NULL.

2. عرض قواعد البيانات الموجودة (كل السيرفر)

الأمر ده شوفناه قبل كده (في الفصول اللي فاتت) وبيوريك كل قواعد البيانات اللي بيديرها السيرفر:

المثال:

mysql> SHOW DATABASES;

3. معرفة الجداول في قاعدة البيانات الحالية (إيه اللي جوا الداتا بيز دي؟)

لو عايز تعرف إيه الجداول اللي عملتها جوه قاعدة البيانات اللي إنت مختارها حالياً (مثلاً menagerie)، بتستخدم أمر SHOW TABLES.

المثال:

mysql> SHOW TABLES;
+---------------------+
| Tables_in_menagerie |
+---------------------+
| event               |
| pet                 |
+---------------------+
  • الشرح: العمود اللي في النتيجة دايماً بيكون اسمه غريب شوية Tables_in_db_name، واسم db_name بيتغير حسب اسم قاعدة البيانات اللي إنت بتسأل عنها.
  • في المثال ده، إحنا في قاعدة بيانات menagerie، وفيها جدولين اسمهم pet و event.

4. معرفة هيكل الجدول (الـ Structure)

ده أهم جزء، وهو اللي بيوريك كل تفصيلة عن الجدول. لو نسيت اسم عمود، أو نوع بياناته، بتستخدم أمر DESCRIBE.

المثال:

mysql> DESCRIBE pet;
+---------+-------------+------+-----+---------+-------+
| Field   | Type        | Null | Key | Default | Extra |
+---------+-------------+------+-----+---------+-------+
| name    | varchar(20) | YES  |     | NULL    |       |
| owner   | varchar(20) | YES  |     | NULL    |       |
| species | varchar(20) | YES  |     | NULL    |       |
| sex     | char(1)     | YES  |     | NULL    |       |
| birth   | date        | YES  |     | NULL    |       |
| death   | date        | YES  |     | NULL    |       |
+---------+-------------+------+-----+---------+-------+

شرح الأعمدة في الـ DESCRIBE:

  1. Field (الحقل): ده اسم العمود.
  2. Type (النوع): ده نوع البيانات اللي متخزنة في العمود ده (زي varchar(20) أو date).
  3. Null (هل يقبل قيم مجهولة؟): بيوريك لو العمود ممكن يسجل فيه قيمة NULL (قيمة مجهولة) ولا لازم يكون فيه بيانات.
  4. Key (مفتاح): لو العمود ده عليه فهرس (Index) أو هو مفتاح أساسي (Primary Key)، بيظهر هنا (زي PRI أو MUL).
  5. Default (القيمة الافتراضية): لو مكنتش دخلت قيمة للعمود، القيمة دي هي اللي هتتحط فيه.
  6. Extra (إضافات): هنا بتظهر معلومات خاصة زي مثلاً لو العمود ده متظبط على خاصية AUTO_INCREMENT (اللي بتزود الرقم تلقائي)، بتظهر هنا auto_increment.

نصيحة سريعة: ممكن تختصر DESCRIBE وتكتب DESC بس عشان تجيب نفس النتيجة.

5. أدوات متقدمة للحصول على معلومات (للمحترفين شوية)

  1. عرض أمر الإنشاء (SHOW CREATE TABLE): لو عايز تعرف الأمر الأصلي بتاع CREATE TABLE اللي اتكتب عشان يعمل الجدول ده، عشان لو عايز تنقله أو تنسخه، بتستخدم الأمر ده.

  2. معلومات الفهارس (SHOW INDEX FROM): لو الجدول بتاعك عليه فهارس (Indexes) عشان تسرع البحث، ممكن تعرف كل تفاصيل الفهارس دي باستخدام أمر SHOW INDEX FROM tbl_name.

الفصل الخامس ده عامل زي فريق الإسعافات الأولية اللي بتنادي عليه لما بتنسى تفاصيل الجدول بتاعك. بدل ما تقعد تحاول تخمن إيه اسم العمود أو إيه نوع بياناته، بتستخدم أوامر SHOW و DESCRIBE عشان تاخد نظرة سريعة وموثوقة على كل حاجة متخزنة في السيرفر بتاعك.
